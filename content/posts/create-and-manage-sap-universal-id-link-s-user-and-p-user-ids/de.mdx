# SAP Universal ID erstellen und verwalten: S-User- und P-User-IDs verknüpfen


### Noel D. Costa

*   Letztes Update : 16\. März 2025

Als ich neu bei SAP war, verbrachte ich mehrere Wochen damit, das Konzept der SAP Universal IDs, S-User-Rollen und P-User-Rollen und deren Zusammenspiel zu verstehen. Vor der SAP Universal ID war es ein Albtraum, den Überblick über all diese Konten zu behalten.

SAP Universal ID löst ein echtes Problem für Berater und Kunden. Es ermöglicht Ihnen [Verwaltung mehrerer SAP](https://noeldcosta.com/de/start-your-sap-implementation-project-right/) Konten unter einem Login. Keine Passwort-Tabellen oder ständigen E-Mails zum Zurücksetzen mehr, wenn Sie vergessen, welche Anmeldeinformationen wohin gehören.

Für Berater, die zwischen verschiedenen Clientsystemen wechseln, bedeutet dies eine enorme Zeitersparnis. Sie können schnell zwischen Ihren P-Benutzerkonten wechseln, ohne sich jedes Mal ab- und wieder anmelden zu müssen. Die Zeitersparnis summiert sich schnell, wenn Sie in mehreren Landschaften arbeiten.

S-User-Konten (diese Administratorkonten für [SAP-Support](https://noeldcosta.com/de/best-sap-implementation-strategies-to-avoid-costly-mistakes/) und Lizenzierung) werden ebenfalls mit Ihrer Universal-ID verknüpft. Das erleichtert Ihnen das Einreichen von Support-Tickets oder das Herunterladen von Software.

Die Implementierung ist noch nicht perfekt. Einige ältere [SAP-Systeme integrieren sich nicht](https://noeldcosta.com/de/sap-implementation-cost-and-budget-breakdown/) reibungslos. Und die Ersteinrichtung kann verwirrend sein, wenn Sie viele bestehende Konten haben.

Aber insgesamt, Universal [ID behebt eine der größten Frustrationen des Alltags](https://noeldcosta.com/de/5-best-crm-systems-for-sap-in-2024/) im SAP-Ökosystem. Wenn Sie mehrere Mandanten oder Systeme verwalten, sollte die Einrichtung ganz oben auf Ihrer To-do-Liste stehen. Die Konfiguration dauert etwa 15 Minuten und erspart Ihnen später unzählige Stunden lästiger Anmeldeprobleme.

SAP Universal ID vereinfacht Ihre [SAP-Erfahrung](https://noeldcosta.com/de/sap-job-description-generator/) Verbinden Sie alle Ihre Konten mit einem einzigen Login. Das spart Zeit und Ärger, da Sie sich nicht mehrere Anmeldeinformationen merken müssen und gleichzeitig die spezifischen Berechtigungen jedes verknüpften Kontos erhalten bleiben.

## Grundlegendes zu SAP-Benutzertypen

Ich habe jahrelang mit verschiedenen SAP-ID-Typen jongliert. Jeder hat seinen eigenen Zweck, und die Kenntnis der Unterschiede ist wichtig, um den richtigen Zugriff zu erhalten.

**P-Benutzer-ID** Ihr [persönliches SAP](https://noeldcosta.com/de/ai-insights-shiftgearx-noeldcosta/) Identität. Sie bleibt bei Ihnen, nicht bei Ihrem Unternehmen. Sie nutzen sie für die SAP-Community, laden Testversionen herunter und posten in Foren. Meine begleitet mich seit 16 Jahren (glaube ich) durch verschiedene Jobs. Es ist quasi Ihr persönlicher SAP-Pass.

**S-Benutzer-ID** funktioniert völlig anders. Es gehört Ihrem Unternehmen, nicht Ihnen. Diese ID bringt Sie in lizenzierte [SAP-Dienstleistungen](https://noeldcosta.com/de/best-sap-documentation-tools-2024-guide/), Support-Tickets und Downloads, die Ihr Unternehmen bezahlt. Kündigen Sie Ihren Job, verlieren Sie den S-User. Ich hatte sechs verschiedene, während ich zwischen Unternehmen wechselte.

[SAP Universal ID behebt endlich](https://noeldcosta.com/de/why-sap-data-migration-fails-and-how-to-fix-it/) Dieses Chaos. Es verbindet Ihren P-User mit mehreren S-Usern unter einem Login. Ein Passwort statt fünf oder sechs. Wechseln Sie das Unternehmen? Verknüpfen Sie Ihren neuen S-User einfach mit Ihrer Universal ID. Das hat mir jede Menge Ärger erspart, als ich in derselben Woche mit verschiedenen Client-Systemen gearbeitet habe.

## Schritt-für-Schritt-Anleitung zum Erstellen Ihrer SAP Universal ID

## \# Schritt 1: Richten Sie Ihre Universal-ID ein

Wenn Sie Ihre SAP Universal ID einrichten müssen, gehen Sie wie folgt vor:

*   Gehen Sie zur SAP Universal ID-Site. Gehen Sie zu [account.sap.com](https://account.sap.com)Sie gelangen auf die Hauptanmeldeseite von SAP. Hier beginnt alles.


*   Klicken Sie auf **"Registrieren"** Es ist gut sichtbar und man kann es nicht übersehen. Wenn Sie bereits eine Art von [SAP-Konto](https://noeldcosta.com/de/sap-quality-gates-implementation/), werden Ihnen möglicherweise andere Optionen angezeigt, aber diese Schaltfläche sollte weiterhin vorhanden sein.


*   Geben Sie Ihre persönlichen Daten ein. Sie müssen Folgendes angeben:  
    *   Vor-und Nachname
    *   E-Mail-Adresse (verwenden Sie eine, die Sie langfristig behalten, keine temporäre Firmen-E-Mail-Adresse)
    *   Erstellen Sie ein sicheres Passwort (muss mindestens 8 Zeichen lang sein und Groß- und Kleinbuchstaben, Zahlen und Sonderzeichen enthalten)
    *   Ihrem Wohnsitzland
    *   Firmenname (optional, aber hilfreich)
*   Akzeptieren [AGB](https://noeldcosta.com/de/sap-negotiation-advisors-reduce-cost/) und Bedingungen Es gibt ein Kontrollkästchen, um die Bedingungen von SAP zu akzeptieren. Sie müssen dieses anklicken, um fortzufahren. Es gibt auch ein optionales Kontrollkästchen für Marketing [Kommunikation](https://noeldcosta.com/de/2025-the-year-sap-generative-ai-redefines-middle-east-careers/). Das ist Ihre Entscheidung.
*   Bestätigen Sie Ihre E-Mail-Adresse. SAP sendet Ihnen einen Bestätigungscode an die angegebene E-Mail-Adresse. Überprüfen Sie Ihren Posteingang (und ggf. Ihren Spam-Ordner). Geben Sie diesen Code innerhalb von 10 Minuten auf der Bestätigungsseite ein, andernfalls müssen Sie einen neuen Code anfordern. Verwenden Sie Ihre persönliche E-Mail-Adresse, nicht die Ihres Unternehmens.
*   Melden Sie sich mit Ihrer neuen Universal-ID an. Nach der Überprüfung werden Sie zur Anmeldung mit Ihren neuen Anmeldedaten weitergeleitet. Ihre Universal-ID ist jetzt aktiv.
*   verbindung [bestehende SAP](https://noeldcosta.com/de/mastering-sap-implementation-a-step-by-step-guide-for-2025/) Konten Nach der Anmeldung werden Ihnen Optionen zum Verknüpfen bestehender Konten angezeigt. Wenn Sie P-User- oder S-User-Konten haben, klicken Sie auf „Konto verknüpfen“ und folgen Sie den Anweisungen. Sie benötigen die Benutzernamen und Passwörter aller zu verknüpfenden Konten.

Der gesamte Vorgang dauert etwa 10 Minuten. Ich empfehle, ihn durchzuführen, wenn Sie Zugriff auf alle Ihre SAP-Kontoinformationen haben, damit Sie alles auf einmal verknüpfen können.

Denken Sie daran, dass Ihre Universal-ID Sie während Ihrer gesamten Karriere begleitet. Verwenden Sie daher nach Möglichkeit eine private E-Mail-Adresse statt einer geschäftlichen.

## \# Schritt 2: Verknüpfen von S-User- und P-User-IDs mit der Universal-ID

Nachdem Sie Ihre Universal ID eingerichtet haben, müssen Sie sie nun mit Ihren S-User- oder P-User-Konten verknüpfen. Wie geht das?

*   Melden Sie sich bei Ihrer SAP Universal ID an. Gehen Sie zu [account.sap.com](https://account.sap.com) und melden Sie sich mit Ihren Universal ID-Anmeldeinformationen an. **Stellen Sie sicher, dass Sie die E-Mail-Adresse und das Passwort verwenden, die Sie während der Einrichtung erstellt haben.**


*   Navigieren Sie zu „Kontoeinstellungen“ > „Verknüpfte Konten“. Suchen Sie nach der Anmeldung oben rechts nach Ihrem Profilsymbol. Klicken Sie darauf, um ein Dropdown-Menü zu öffnen, und wählen Sie dann „Kontoeinstellungen“. In der linken Seitenleiste sehen Sie die Option „Verknüpfte Konten“. Klicken Sie darauf.
*   Geben Sie die Anmeldedaten Ihres S- oder P-Benutzers ein. Auf der Seite „Verknüpfte Konten“ werden die aktuell verknüpften Konten (sofern vorhanden) angezeigt und Sie können weitere hinzufügen. Klicken Sie auf „Konto verknüpfen“. Ein Popup-Fenster wird angezeigt und fragt nach Benutzername und Passwort des zu verknüpfenden Kontos.
*   Bestätigen und verknüpfen Klicken Sie nach Eingabe der Anmeldeinformationen auf „Konto verknüpfen“. [Das System überprüft die Informationen](https://noeldcosta.com/de/my-journey-with-customer-information-solutions-defense/). Bei erfolgreichem Abschluss wird eine Bestätigungsmeldung angezeigt. Das Konto wird nun in der Liste Ihrer verknüpften Konten angezeigt.
*   Wiederholen Sie die Schritte 3 und 4 für jedes weitere Konto, das Sie verknüpfen möchten. Es gibt keine Begrenzung für die Anzahl der Konten, die Sie mit Ihrer Universal ID verknüpfen können.

**Ich empfehle Ihnen, alle Ihre [Kontoanmeldeinformationen bereit, bevor Sie beginnen](https://noeldcosta.com/de/5-best-crm-systems-for-sap-in-2024/) Dieser Vorgang. Notieren Sie sich, welche Konten Sie verknüpft haben, da Sie leicht einige der älteren S-Benutzer-IDs vergessen können, wenn Sie mit mehreren SAP-Kunden gearbeitet haben.**

Nach der Verknüpfung können Sie mit Ihrer Universal-ID auf alle diese Konten zugreifen. Das spart enorm viel Zeit.

## So erhalten Sie eine S-User-ID

S-Benutzer-IDs werden von Ihrem Unternehmen verwaltet. Im Gegensatz zu P-Benutzern können Sie keine selbst erstellen. Diese Konten gehören Ihrem Unternehmen, nicht Ihnen.

Um einen S-User zu erhalten, [Fordern Sie eines über Ihr SAP an](https://noeldcosta.com/de/sap-project-scope-template-management-and-control/) Superadministrator. Diese Person [verwaltet alle S-User-Konten](https://noeldcosta.com/de/ai-governance-framework-guide-building-a-responsible-ai-plan/) für Ihre Organisation. Wenn Sie nicht wissen, wer diese Rolle innehat, fragen Sie Ihre IT-Abteilung. In kleineren Unternehmen ist es in der Regel der IT-Direktor oder [SAP Project Manager](https://noeldcosta.com/de/project-planning-and-control-get-sap-projects-back-on-track/).

Für [SAP Super Admins, der Erstellungsprozess](https://noeldcosta.com/de/best-sap-implementation-templates-activate-2024/) ist ganz einfach. Sie melden sich mit ihrem Administrator-S-Benutzer beim SAP Support Portal an, gehen zu „Benutzerverwaltung“ und wählen „S-Benutzer erstellen“. 

**Sie benötigen Ihren Namen, Ihre E-Mail-Adresse und Ihre Telefonnummer. Das System generiert Ihren Benutzernamen automatisch.**

Rollenzuweisungen sind entscheidend. Ihr Super-Admin gibt genau an, auf was Sie zugreifen können:

*   Support (zum Erstellen von Tickets)
*   Lizenz (für die Lizenzverwaltung)
*   Download (für Software-Downloads)
*   Systeme (zur Systemregistrierung)

Most [Benutzer benötigen Support](https://noeldcosta.com/de/change-management-plan-success/) und Download-Rollen. Lizenzrollen sind normalerweise auf wenige Personen beschränkt.

Nach der Erstellung erhalten Sie eine E-Mail mit den Aktivierungsschritten. Wenn Sie das Unternehmen verlassen, wird Ihr S-User gekündigt, da er dem Unternehmen und nicht Ihnen gehört.


## So erhalten Sie eine P-Benutzer-ID

Ich habe meinen P-Benutzer jetzt seit über einem Jahrzehnt. Im Gegensatz zu S-Benutzern sind P-Benutzer-IDs persönliche Konten, die Ihnen und nicht Ihrem Arbeitgeber gehören.

Die Einrichtung eines P-Users ist ganz einfach. Besuchen Sie die SAP Community (community.sap.com) und klicken Sie auf „Registrieren“. Sie benötigen eine gültige E-Mail-Adresse. Ich empfehle Ihnen dringend, Ihre private E-Mail-Adresse anstelle der geschäftlichen zu verwenden, da Sie dieses Konto während Ihrer gesamten Karriere begleiten wird.

Das Registrierungsformular fragt nach grundlegenden Informationen wie Name, Standort und beruflichen Daten. Nach dem Absenden erhalten Sie eine Bestätigungs-E-Mail. Klicken Sie auf den Link, und schon kann es losgehen.

P-Benutzer geben Ihnen Zugriff auf:

*   [Foren und Inhalte der SAP-Community](https://noeldcosta.com/de/write-for-us-lets-share-our-experiences/)
*   Kostenlose Testsysteme
*   SAP-Entwicklerzentrum
*   [Übungsmaterial](https://noeldcosta.com/de/sap-solution-builder/)
*   Veranstaltungsregistrierung

Wichtig zu beachten: P-User bieten keinen Zugriff auf lizenzierten Support oder Downloads. Dafür sind S-User zuständig. Ihr P-User bleibt Ihnen jedoch unabhängig von Jobwechseln erhalten.

Ich nutze mein Konto ständig für Community-Fragen, zum Lesen von Blogs und für den Zugriff auf Entwicklerressourcen. Es ist im Grunde Ihre persönliche Identität im SAP-Ökosystem.

Wenn Sie später Ihre Universal-ID einrichten, verknüpfen Sie diesen P-Benutzer damit für den Single-Sign-On-Zugriff.

## Verwalten verknüpfter Konten

Ich habe mehrere SAP-Konten mit meiner Universal ID verknüpft. Die ordnungsgemäße Verwaltung erspart mir jede Menge Ärger.

*   **Das Wechseln zwischen verknüpften Konten ist ziemlich einfach****.** 
    *   Nachdem Sie sich mit Ihrer Universal-ID angemeldet haben, klicken Sie oben rechts auf Ihr Profilbild. Es wird eine Dropdown-Liste mit allen verknüpften Konten angezeigt.
    *   Klicken Sie einfach auf die gewünschte Option. Das System wechselt den Kontext, ohne dass Sie sich erneut anmelden müssen. Ich verwende diese Funktion ständig, wenn ich zwischen Client-Umgebungen wechsle.
*   **Die Verwendung der richtigen ID für verschiedene SAP-Dienste ist entscheidend. Einige Dienste funktionieren nur mit bestimmten Kontotypen.** Für Support-Tickets und Downloads ist ein S-User mit den entsprechenden Berechtigungen erforderlich. Community-Beiträge verwenden Ihre P-User-Identität. Bei Verwendung des falschen ID-Typs treten frustrierende Zugriffsfehler auf.
*   **Die Verwaltung von Berechtigungen für S-User erfordert die Zusammenarbeit mit den [SAP](https://noeldcosta.com/de/best-sap-implementation-templates-activate-2024/) Administrator****.** Ihr Administrator steuert Ihren Zugriff über den Arbeitsbereich „Benutzerverwaltung“ im SAP Support Portal. Wenn Sie zusätzliche Zugriffsrechte benötigen (z. B. für Lizenzverwaltung oder Systemregistrierung), müssen Sie diese bei Ihrem Administrator anfordern. Sie können nicht über Ihre Universal ID geändert werden.

Denken Sie daran: Die Universal ID bietet zwar den Komfort einer einmaligen Anmeldung, ändert jedoch nichts an den zugrunde liegenden Berechtigungen der einzelnen Konten. Jedes verknüpfte Konto verfügt weiterhin über eigene Zugriffsrechte.

## Häufige Probleme und Lösungen

Ich bin im Laufe der Jahre auf zahlreiche SAP-ID-Probleme gestoßen. Hier sind Lösungen für die häufigsten Probleme:

### 1\. SAP Admin nicht gefunden – so fordern Sie Hilfe an

Wenn Sie nicht herausfinden können, wer Ihr SAP-Administrator ist, wenden Sie sich an Ihre IT-Abteilung. Diese weiß in der Regel, wer [verwaltet SAP](https://noeldcosta.com/de/2024-sap-timeline-planning-implementation-guide-essentials/) Zugang. Fragen Sie in kleineren Unternehmen Ihren [SAP-Projektmanager](https://noeldcosta.com/de/how-to-create-an-sap-implementation-project-charter/) oder IT-Leiter. Wenn das nicht funktioniert, wenden Sie sich an Ihr Finanzteam. Dieses verfügt in der Regel aus Lizenzgründen über S-User-Zugriff und weiß, wer der Administrator ist.

Als letzte Möglichkeit können Sie SAP direkt über die allgemeinen Supportkanäle kontaktieren. Dort können Sie die Kontaktdaten Ihres Super-Admins einsehen, erhalten aber keinen direkten Zugriff.

### 2\. Verknüpfungsfehler – Anmeldeinformationen stimmen nicht überein, Kontosperren

Wenn die Kontoverknüpfung fehlschlägt, überprüfen Sie zunächst Ihre Anmeldeinformationen. Tippfehler sind die häufigsten Ursachen. Wenn Sie sicher sind, dass Benutzername und Passwort korrekt sind, kann Ihr Konto nach zu vielen Fehlversuchen gesperrt werden. Das passiert mir mindestens einmal im Jahr.

Bei gesperrten Konten müssen Sie sich an den SAP-Administrator Ihres Unternehmens wenden (S-Benutzer) oder die Funktion zum Zurücksetzen des Passworts (P-Benutzer) nutzen. Warten Sie nach dem Zurücksetzen etwa 15 Minuten, bevor Sie erneut versuchen, die Verbindung herzustellen – die Systeme synchronisieren nicht sofort.

Manchmal verursachen Konten mit identischen E-Mail-Adressen Probleme. Wenn Sie mehrere Konten mit derselben E-Mail-Adresse erstellt haben, müssen Sie sich an den SAP-Support wenden, um die Konten zu entwirren.

### 3\. E-Mail nicht verifiziert – Fehlerbehebung

Bestätigungs-E-Mails gehen manchmal verloren oder landen im Spam-Ordner. Überprüfen Sie zuerst Ihren Spam-Ordner. Wenn Sie die E-Mail nach 10 Minuten nicht erhalten haben, verwenden Sie die Option „Bestätigung erneut senden“.

Wenn Sie bei der Registrierung eine falsche E-Mail-Adresse angegeben haben, müssen Sie sich erneut mit der richtigen Adresse registrieren. Eine nicht verifizierte E-Mail-Adresse kann nicht geändert werden.

Wenn Bestätigungslinks nicht funktionieren, kopieren Sie die URL direkt in Ihren Browser, anstatt darauf zu klicken. Einige E-Mail-Clients beschädigen die Linkformatierung.

## SAP Universal ID: Häufige Fehler und Fehlerbehebungen (vollständige Liste)

Wenn Sie versucht haben, Konten über die SAP Universal ID zu verknüpfen, sind Ihnen möglicherweise Fehler begegnet, die zunächst keinen Sinn ergaben. Möglicherweise hat das System Ihren S-User blockiert, oder Sie haben eine Warnung wegen einer falschen E-Mail-Adresse erhalten, obwohl alles korrekt aussah. 

Die SAP Universal ID soll den Zugriff vereinfachen. Dennoch gestaltet sich der Verknüpfungsprozess oft schwieriger als erwartet. Ich habe schon erlebt, dass die Verbindung zu Konten aufgrund einer nicht verifizierten E-Mail-Adresse fehlschlug. Diese Fehler treten häufig auf und ihre Behebung erfordert in der Regel einige manuelle Schritte. 

Nachfolgend finden Sie eine Aufschlüsselung der häufigsten Probleme und Informationen zu deren Lösung.

### 1\. „Diese Benutzer-ID ist bereits mit einer anderen SAP Universal ID verknüpft“

*   **Verursachen**: Der S-Benutzer oder P-Benutzer ist bereits an eine andere SAP Universal ID gebunden.

*   **Auflösung**:

    *   Melden Sie sich bei der Universal-ID an, mit der es bereits verknüpft ist.

    *   Navigieren **Kontoeinstellungen** > **Verbundene Konten**.

    *   Heben Sie dort die Verknüpfung des Kontos auf.

    *   Versuchen Sie die Verknüpfung erneut mit Ihrer gewünschten Universal-ID.

    *   Wenn die ursprüngliche Universal-ID nicht zugänglich ist, erstellen Sie ein Ticket unter Komponente `XX-SER-SAPSMP-USR`.

### 2\. „Sie sind nicht berechtigt, diese Benutzer-ID zu verknüpfen“

*   **Verursachen**: Sie versuchen, eine ID zu verknüpfen, für deren Verwaltung Sie keine Berechtigung haben (falsches Unternehmen oder fehlende Berechtigungen).

*   **Auflösung**:

    *   Bestätigen Sie, dass Sie der rechtmäßige Eigentümer oder Benutzer des S-Benutzers/P-Benutzers sind.

    *   Bitten Sie den SAP-Administrator Ihres Unternehmens, eine Verknüpfung herzustellen oder Zugriff zu gewähren.

    *   Wenn Sie einen Partner-/Kundenbenutzer verknüpfen, stellen Sie sicher, dass dieser sich im Rahmen Ihres Partnernetzwerks befindet.

### 3\. „E-Mail-Adresse stimmt nicht überein – Benutzer-ID kann nicht verknüpft werden“

*   **Verursachen**: Die E-Mail des S-Benutzers/P-Benutzers stimmt nicht mit Ihrer Universal ID-E-Mail überein.

*   **Auflösung**:

    *   Richten Sie die E-Mail-Adresse für alle Konten gleich aus.

        *   Aktualisieren Sie Ihre Universal ID-E-Mail unter [accounts.sap.com](https://accounts.sap.com).

        *   Bitten Sie Ihren SAP-Administrator, die E-Mail des S-Benutzers zu aktualisieren über **Die Benutzerverwaltung** im Support-Portal.

### 4\. „Dieser S-Benutzer ist inaktiv oder wurde gelöscht“

*   **Verursachen**: Der S-Benutzer wurde deaktiviert oder ist abgelaufen.

*   **Auflösung**:

    *   Kontaktieren Sie Ihren internen SAP-Administrator.

    *   Reaktivierung oder Ausstellung eines neuen S-Users beantragen.

    *   Administratoren können dies tun über **Kundenservice** > **Die Benutzerverwaltung**.

### 5\. „Technischer Fehler – Bitte versuchen Sie es später noch einmal“

*   **Verursachen**: Backend-Problem oder abgelaufene Sitzung während der Verknüpfung.

*   **Auflösung**:

    *   Melden Sie sich von allen SAP-Plattformen ab.

    *   Browser-Cache/Cookies löschen.

    *   Versuchen Sie es erneut im Inkognito-/Privatmodus.

    *   Wenn das Problem weiterhin besteht, öffnen Sie ein Supportticket bei SAP.

### 6\. „Sie müssen Ihre Universal-ID verwenden, um sich anzumelden.“

*   **Verursachen**: Versuch, nach der Verknüpfung alte S-Benutzer-/P-Benutzer-Anmeldeinformationen zu verwenden.

*   **Auflösung**:

    *   Melden Sie sich immer über **Universal ID-E-Mail und Passwort** at [login.sap.com](https://login.sap.com).

    *   Vermeiden Sie die direkte Verwendung von S-User-Anmeldeinformationen, sobald eine Verknüpfung hergestellt wurde.

### 7\. „Zu viele Versuche. Konto vorübergehend gesperrt.“

*   **Verursachen**: Mehrere fehlgeschlagene Anmelde- oder Verknüpfungsversuche haben eine Sperre ausgelöst.

*   **Auflösung**:

    *   Warten Sie 15–30 Minuten auf die automatische Entsperrung.

    *   Alternativ können Sie das Passwort zurücksetzen über accounts.sap.com > Passwort vergessen.

### 8\. „Diesem S-Benutzer sind keine gültigen Rollen zugewiesen“

*   **Verursachen**: Der S-User ist vorhanden, hat aber keine zugewiesenen Berechtigungen (z. B. Support-Zugriff).

*   **Auflösung**:

    *   Wenden Sie sich an Ihren SAP-Administrator.

    *   Zuweisen der erforderlichen Rollen mithilfe von **Die Benutzerverwaltung** im Support-Portal.

    *   Empfohlenes Minimum: Vorfälle anzeigen, Profil bearbeiten, Software herunterladen.

### 9\. „Ihrer SAP Universal ID fehlt ein zweiter E-Mail-Link!“

*   **Verursachen**: Sie haben eine zweite E-Mail-Adresse hinzugefügt, diese aber nicht bestätigt.

*   **Auflösung**:

    *   Einloggen in [accounts.sap.com](https://accounts.sap.com).

    *   Gehe zu **Personenbezogene Daten** > **E-mailadressen**.

    *   Senden Sie die Bestätigungs-E-Mail erneut.

    *   Klicken Sie auf den Link in der Bestätigungs-E-Mail, um den Vorgang abzuschließen.

### 10\. „Ihre Sitzung ist abgelaufen“ / „Ungültiges Token“

*   **Verursachen**: Sitzungstimeout oder veraltetes Authentifizierungstoken.

*   **Auflösung**:

    *   Melden Sie sich von allen SAP-Systemen ab.

    *   Browser-Cache löschen.

    *   Melden Sie sich noch einmal von vorne an und wiederholen Sie den Verknüpfungsvorgang.

### 11\. „Sie versuchen, eine Benutzer-ID aus einer anderen Region/einem anderen Markt zu verknüpfen.“

*   **Verursachen**: Versuch, eine Benutzer-ID aus einer anderen SAP-Entität oder -Region zu verknüpfen.

*   **Auflösung**:

    *   Stellen Sie eine Supportanfrage bei SAP, um den Link manuell zu genehmigen.

    *   Bestätigen Sie die Gültigkeit des regionalen Kontos.

    *   Dies passiert manchmal, wenn Sie SAP Business Network- oder Ariba-IDs von außerhalb Ihrer primären Region verknüpfen.

### 12\. „Die Benutzer-ID, die Sie verknüpfen möchten, hat nicht das richtige Format.“

*   **Verursachen**: Eingabe einer ungültigen ID (z. B. Lieferanten-ID, SCN-ID, Kundennummer).

*   **Auflösung**:

    *   Einziger gültiger Link:

        *   **S-Benutzer** (z.B, `S0001234567`)

        *   **P-Benutzer** (z.B, `P12345678`)

    *   Vermeiden Sie die Eingabe von Geschäftspartnernummern oder veralteten SAP-IDs.

### 13\. „Kontoverknüpfungslimit erreicht“

*   **Verursachen**: SAP Universal ID hat ein Limit (normalerweise 10) für verknüpfte Konten.

*   **Auflösung**:

    *   Gehe zu **Verbundene Konten** at [accounts.sap.com](https://accounts.sap.com).

    *   Die Verknüpfung von Konten wird nicht mehr benötigt.

    *   Fahren Sie dann mit der Verknüpfung der neuen Benutzer-ID fort.

### 14\. So trennen Sie die Verknüpfung von S-User oder P-User mit der SAP Universal ID

**Schritte:**

1.  Gehe zu [accounts.sap.com](https://accounts.sap.com)

2.  Melden Sie sich mit Ihrer Universal-ID an.

3.  Klicken Sie auf **Kontoeinstellungen** > **Verbundene Konten**.

4.  Suchen Sie den S-Benutzer oder P-Benutzer, den Sie entfernen möchten.

5.  Klicken Sie auf **Verknüpfung aufheben / entfernen**.

6.  Bestätigen.

**Wichtig:**  
Wenn Sie die Verknüpfung nicht aufheben können (z. B. weil der Zugriff auf die Universal-ID verloren gegangen ist), erstellen Sie ein Support-Ticket unter Komponente `XX-SER-SAPSMP-USR`.

## Fazit

Ich benutze SAP Universal ID seit der Einführung. [Verwaltung aller meiner SAP](https://noeldcosta.com/de/sap-enterprise-warehouse-management-sap-ewm-essentials/) Logins viel einfacher. Ein Login für alles, kein Jonglieren mit Passwörtern mehr, einfacher Wechsel zwischen Systemen.

Für [Berater, die zwischen Kunden wechseln, oder jeder, der verschiedene SAP-](https://noeldcosta.com/de/best-sap-technical-change-management-tools-2025/) Mit Universal ID sparen Sie jede Menge Zeit. Wahrscheinlich sparen Sie mir täglich 10–15 Minuten, da ich nicht nach Passwörtern suchen oder Konten zurücksetzen muss, die ich vergessen habe.

Die S-User-Verwaltung ist jedoch weiterhin wichtig. Diese Unternehmenskonten steuern den Zugriff auf lizenzierte Inhalte und Support. Unternehmen zahlen viel Geld für SAP, also kontrollieren sie, wer Zugriff erhält.

Wenn Sie Hilfe benötigen, wenden Sie sich zunächst an den SAP-Administrator Ihres Unternehmens. Dieser kümmert sich um die meisten S-User-Probleme und kennt die Konfiguration Ihres Unternehmens. Bei technischen Problemen mit der Universal ID bietet das SAP-Hilfeportal umfassende Dokumentation und Support.

Die Einrichtung dauert etwa 10 Minuten, spart aber unzählige Stunden. Wenn Sie SAP regelmäßig nutzen, tun Sie es einfach.

Universal ID ist besonders nützlich bei Systemänderungen oder neuen [SAP-Rollouts](https://noeldcosta.com/de/creating-an-effective-sap-project-steering-committee/)Anstatt für alles neue Logins zu benötigen, verknüpfen Sie diese einfach mit Ihrer Universal-ID. Einfacher für alle.

Denken Sie jedoch daran: Die Universal-ID hilft beim Anmelden, ändert aber nicht die Berechtigungen. Jedes Konto behält seine spezifischen Zugriffsrechte.

Aus Erfahrung, bekommen Sie Ihre [SAP](https://noeldcosta.com/de/sap-ecc-to-s4hana-migration-case-study/) Die Organisation von Identitäten zahlt sich aus. Sie sorgt für einen reibungsloseren Ablauf, egal, ob Sie SAP einmal pro Woche oder täglich den ganzen Tag nutzen.


Wenn Sie Fragen haben oder eine Situation besprechen möchten, die Sie in Ihrem [SAP-Implementierung](https://noeldcosta.com/de/sap-license-negotiation-10-key-points-to-consider-in-2024/), zögern Sie bitte nicht, sich zu melden!

## Fragen, die Sie möglicherweise haben ...

### 1\. Was ist die SAP Universal ID und warum sollte ich sie verwenden?

Die SAP Universal ID ist ein einheitlicher Login, der alle Ihre SAP-Konten verbindet. Sie ermöglicht Ihnen den Zugriff auf mehrere SAP-Systeme, ohne mit unterschiedlichen Passwörtern und Benutzernamen jonglieren zu müssen.

### 2\. Kann ich sowohl P-User- als auch S-User-Konten mit meiner SAP Universal ID verknüpfen?

Ja, Sie können sowohl persönliche P-User- als auch Firmen-S-User-Konten mit Ihrer Universal ID verknüpfen. Das Wechseln zwischen den Konten ist kinderleicht, ohne dass Sie sich ab- und wieder anmelden müssen.

### 3\. Wird mein Unternehmen davon erfahren, wenn ich eine SAP Universal ID erstelle?

Die Erstellung einer Universal-ID ist Ihre persönliche Angelegenheit und unabhängig von Ihrem Unternehmen. Wenn Sie jedoch Unternehmens-S-User verknüpfen, können Ihre SAP-Administratoren diese Aktivitäten weiterhin sehen.

### 4\. Verliere ich meine SAP Universal ID, wenn ich mein Unternehmen verlasse?

Nein, Ihre SAP Universal ID bleibt bei Ihnen. Sie verlieren zwar den Zugriff auf die S-User Ihres Unternehmens, Ihre Universal ID und alle P-User bleiben jedoch weiterhin bei Ihnen.

### 5\. Kann ich die E-Mail-Adresse für meine SAP Universal ID ändern?

Ja, öffne einfach deine Kontoeinstellungen und aktualisiere sie. Vergiss nicht, die neue E-Mail-Adresse nach der Änderung zu bestätigen, sonst wirst du gesperrt.

### 6\. Wie viele Konten kann ich mit meiner SAP Universal ID verknüpfen?

Keine wirkliche Begrenzung. Ich habe mehrere S-Benutzer von verschiedenen Kunden sowie ein paar P-Benutzer, die alle mit meinem verknüpft sind.

### 7\. Was passiert, wenn das Passwort für ein verknüpftes Konto abläuft?

Bei S-Usern mit abgelaufenen Passwörtern müssen Sie sich an den SAP-Administrator Ihres Unternehmens wenden. Bei P-Usern nutzen Sie einfach die Reset-Funktion.

### 8\. Gibt mir die SAP Universal ID mehr Berechtigungen in SAP-Systemen?

Überhaupt nicht. Es erleichtert lediglich die Anmeldung. Ihre tatsächlichen Berechtigungen hängen weiterhin von jedem einzelnen Konto ab.

### 9\. Wie trenne ich ein Konto von meiner SAP Universal ID?

Gehen Sie zu „Kontoeinstellungen“, dann zu „Verknüpfte Konten“, suchen Sie das Konto, das Sie entfernen möchten, und klicken Sie auf „Verknüpfung aufheben“. Ziemlich unkompliziert.

### 10\. Ist für den Zugriff auf SAP-Systeme eine SAP Universal ID erforderlich?

Es ist optional, aber es lohnt sich. Sie können weiterhin individuelle Anmeldeinformationen verwenden, wenn Sie möchten. Aber warum sollten Sie es sich schwerer machen?
