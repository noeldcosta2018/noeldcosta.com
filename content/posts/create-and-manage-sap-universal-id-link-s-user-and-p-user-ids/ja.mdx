# SAPユニバーサルIDの作成と管理：SユーザーIDとPユーザーIDのリンク


### ノエル・D・コスタ

*   最後の更新 ： 2025 年 3 月 16 日

私が SAP を使い始めたばかりの頃、SAP ユニバーサル ID、S ユーザー ロール、P ユーザー ロールの概念と、それらがどのように連携するかを理解するのに数週間を費やしました。SAP ユニバーサル ID が登場する前は、これらすべてのアカウントを追跡するのはまさに悪夢でした。

SAPユニバーサルIDは、コンサルタントや顧客にとっての悩みの種を解決します。 [複数のSAPを管理する](https://noeldcosta.com/ja/start-your-sap-implementation-project-right/) 1 回のログインで複数のアカウントを管理できます。パスワードのスプレッドシートや、どの資格情報がどこに保存されているか忘れた場合の頻繁なリセット メールは不要になります。

クライアント システム間を行き来するコンサルタントにとって、これは大きな時間節約になります。毎回ログアウトしてログインし直すことなく、P ユーザー アカウント間をすばやく切り替えることができます。複数の環境で作業する場合、時間の節約はすぐに積み重なっていきます。

Sユーザーアカウント（ [SAP サポート](https://noeldcosta.com/ja/best-sap-implementation-strategies-to-avoid-costly-mistakes/) およびライセンスもユニバーサル ID に接続します。サポート チケットを送信したり、ソフトウェアをダウンロードしたりする必要がある場合に便利です。

実装はまだ完璧ではありません。古い [SAPシステムは統合されない](https://noeldcosta.com/ja/sap-implementation-cost-and-budget-breakdown/) スムーズに。既存のアカウントが多数ある場合、初期設定が混乱する可能性があります。

しかし全体的には、ユニバーサル [IDは日々の最大のストレスの1つを解決します](https://noeldcosta.com/ja/5-best-crm-systems-for-sap-in-2024/) SAP エコシステムで。複数のクライアントやシステムを同時に管理している場合は、これを設定する必要があります。設定には約 15 分かかり、今後のログインの手間を大幅に省くことができます。

SAPユニバーサルIDは、 [SAPの経験](https://noeldcosta.com/ja/sap-job-description-generator/) すべてのアカウントを 1 回のログインで接続します。リンクされた各アカウントの特定の権限を維持しながら、複数の資格情報を覚えておく必要がなくなるため、時間とストレスが節約されます。

## SAP ユーザー タイプを理解する

私は長年、さまざまな SAP ID タイプを使い分けてきました。それぞれに独自の目的があり、その違いを理解することが適切なアクセスを得るために重要です。

**P-ユーザーID** あなたです [個人用SAP](https://noeldcosta.com/ja/ai-insights-shiftgearx-noeldcosta/) IDです。会社ではなく、あなたに付き添うものです。SAPコミュニティ、トライアルのダウンロード、フォーラムへの投稿などに使用します。私の場合は、様々な職種を経て16年（たぶん）使い続けています。いわば、あなた専用のSAPパスポートです。

**SユーザーID** 仕組みは全く異なります。それはあなたの会社に属し、あなたに属していません。このIDはライセンスを取得するためのものです [SAP サービス](https://noeldcosta.com/ja/best-sap-documentation-tools-2024-guide/)、サポート チケット、および会社が支払うダウンロード。仕事を辞めると、S ユーザーも失います。私は会社を転々とする中で、6 つの異なる S ユーザーを持っていました。

[SAPユニバーサルIDがついに修正](https://noeldcosta.com/ja/why-sap-data-migration-fails-and-how-to-fix-it/) この混乱を解消します。P ユーザーを 1 回のログインで複数の S ユーザーに接続します。5 つまたは 6 つのパスワードではなく 1 つのパスワードです。会社を切り替えますか? 新しい S ユーザーをユニバーサル ID にリンクするだけです。同じ週に異なるクライアント システムで作業するときに、多くの頭痛の種が解消されました。

## SAP ユニバーサル ID を作成するためのステップバイステップ ガイド

## \# ステップ1: ユニバーサルIDを設定する

SAP Universal ID を設定する必要がある場合は、次の手順に従ってください。

*   SAPユニバーサルIDサイトへ移動 [アカウント](https://account.sap.com)メインの SAP ログイン ページが表示されます。ここからすべてが始まります。


*   詳しくはこちら **"登録"** かなり目立つので見逃すことはありません。すでに何らかの [SAPアカウント](https://noeldcosta.com/ja/sap-quality-gates-implementation/)異なるオプションが表示される場合もありますが、このボタンはそのまま表示されます。


*   個人情報を入力してください。提供する必要がある情報は次のとおりです。  
    *   氏名
    *   メールアドレス（一時的な会社のメールアドレスではなく、長期的に保持するメールアドレスを使用してください）
    *   強力なパスワードを作成してください（少なくとも 8 文字、大文字、小文字、数字、特殊文字を含める必要があります）
    *   お住まいの国
    *   会社名（オプションですが役立ちます）
*   同意する [条件](https://noeldcosta.com/ja/sap-negotiation-advisors-reduce-cost/) および条件 SAPの条件に同意するためのチェックボックスがあります。続行するにはこれをクリックする必要があります。マーケティング用のオプションのチェックボックスもあります。 [コミュニケーション](https://noeldcosta.com/ja/2025-the-year-sap-generative-ai-redefines-middle-east-careers/)それはあなたの判断次第です。
*   メールアドレスを確認してください。SAP は、指定したメールアドレスに確認コードを送信します。受信トレイ (および必要に応じてスパム フォルダ) を確認してください。10 分以内に確認ページでこのコードを入力してください。入力しない場合は、新しいコードをリクエストする必要があります。会社のメール ID ではなく、個人のメール ID を使用してください。
*   新しいユニバーサル ID にログインします。確認されると、新しい認証情報を使用してログインするように指示されます。これで、ユニバーサル ID がアクティブになります。
*   リンク [既存のSAP](https://noeldcosta.com/ja/mastering-sap-implementation-a-step-by-step-guide-for-2025/) アカウント ログイン後、既存のアカウントをリンクするオプションが表示されます。P ユーザーまたは S ユーザー アカウントをお持ちの場合は、\[アカウントをリンク\] をクリックして、指示に従ってください。リンクする各アカウントのユーザー名とパスワードを知っておく必要があります。

全体のプロセスには約 10 分かかります。すべての SAP アカウント情報にアクセスできるときに実行することをお勧めします。そうすれば、すべてを一度にリンクできます。

ユニバーサル ID はキャリアを通じて継続されるため、可能であれば仕事用のメールではなく個人用のメールを使用してください。

## \# ステップ 2: S ユーザー ID と P ユーザー ID をユニバーサル ID にリンクする

ユニバーサル ID を設定したら、次はそれを S ユーザーまたは P ユーザー アカウントにリンクする必要があります。これはどのように行うのですか?

*   SAPユニバーサルIDにログイン [アカウント](https://account.sap.com) ユニバーサル ID 認証情報を使用してサインインします。 **セットアップ時に作成したメールアドレスとパスワードを使用していることを確認してください。**


*   「アカウント設定」>「リンクされたアカウント」に移動します。ログインしたら、右上隅にあるプロフィールアイコンを探します。それをクリックしてドロップダウンメニューを開き、「アカウント設定」を選択します。左側のサイドバーに「リンクされたアカウント」オプションが表示されますので、それをクリックします。
*   S ユーザーまたは P ユーザーの資格情報を入力します。\[リンクされたアカウント\] ページに、現在リンクされているアカウント (ある場合) を表示するセクションと、さらに追加するオプションが表示されます。\[アカウントをリンク\] ボタンをクリックします。リンクするアカウントのユーザー名とパスワードを求めるポップアップが表示されます。
*   確認してリンクする認証情報を入力したら、「アカウントをリンク」をクリックします。 [システムが情報を検証します](https://noeldcosta.com/ja/my-journey-with-customer-information-solutions-defense/)成功すると確認メッセージが表示されます。アカウントがリンク済みアカウントのリストに表示されます。
*   複数のアカウントで繰り返します。リンクする追加アカウントごとに手順 3 ～ 4 を繰り返すことができます。ユニバーサル ID に接続できるアカウントの数に制限はありません。

**すべての [開始する前にアカウント認証情報を用意してください](https://noeldcosta.com/ja/5-best-crm-systems-for-sap-in-2024/) このプロセスでは、複数のSAP顧客と連携している場合は、古いSユーザーIDを忘れてしまう可能性があるため、リンクしたアカウントを記録しておいてください。**

リンクすると、1 つのユニバーサル ID ログインでこれらすべてのアカウントにアクセスできるようになります。これにより、時間を大幅に節約できます。

## SユーザーIDの取得方法

S ユーザー ID は会社によって管理されます。P ユーザーとは異なり、自分で作成することはできません。これらのアカウントは、あなたではなく会社に属します。

Sユーザーになるには、 [SAPを通じてリクエストする](https://noeldcosta.com/ja/sap-project-scope-template-management-and-control/) スーパー管理者。この人物は [すべてのSユーザーアカウントを管理します](https://noeldcosta.com/ja/ai-governance-framework-guide-building-a-responsible-ai-plan/) 組織で誰がこの役割を担っているのかわからない場合は、IT部門に問い合わせてください。小規模な企業では、通常ITディレクターまたは [SAPプロジェクトマネージャー](https://noeldcosta.com/ja/project-planning-and-control-get-sap-projects-back-on-track/).

『Brooklyn Galaxy』のために、倪氏はブルックリン美術館のコレクションからXNUMX点の名品を選び、そのイメージを極めて詳細に描き込みました。これらの作品は、彼の作品とともに中国ギャラリーに展示されています。彼はXNUMX年にこの作品の制作を開始しましたが、最初の硬貨には、当館が所蔵する [SAPスーパー管理者の作成プロセス](https://noeldcosta.com/ja/best-sap-implementation-templates-activate-2024/) 非常に簡単です。管理者SユーザーでSAP Support Portalにログインし、「ユーザー管理」に移動して「Sユーザーの作成」を選択します。 

**名前、メールアドレス、電話番号が必要です。ユーザー名はシステムによって自動的に生成されます。**

役割の割り当ては重要です。スーパー管理者は、アクセスできる内容を正確に指定します。

*   サポート（チケット作成用）
*   ライセンス（ライセンス管理用）
*   ダウンロード（ソフトウェアのダウンロード用）
*   システム（システム登録用）

ブリッジ [ユーザーはサポートを必要としている](https://noeldcosta.com/ja/change-management-plan-success/) およびダウンロード ロール。ライセンス ロールは通常、数人に制限されます。

作成すると、アクティベーション手順が記載されたメールが届きます。退職すると、S ユーザーはあなたではなく会社の所有物となるため、会社側で S ユーザーは解約されます。


## PユーザーIDの取得方法

私はもう 10 年以上 P ユーザーを所有しています。S ユーザーとは異なり、P ユーザー ID は雇用主ではなく、ユーザーに属する個人アカウントです。

Pユーザーの作成は簡単です。SAPコミュニティ（community.sap.com）にアクセスし、「登録」ボタンをクリックしてください。有効なメールアドレスが必要です。このアカウントはキャリアを通して利用されるため、仕事用のメールアドレスではなく個人用のメールアドレスを使用することを強くお勧めします。

登録フォームでは、氏名、住所、職業などの基本情報をご入力いただきます。送信後、確認メールが届きますので、リンクをクリックするだけで登録完了です。

P ユーザーは次のものにアクセスできます:

*   [SAP コミュニティフォーラムとコンテンツ](https://noeldcosta.com/ja/write-for-us-lets-share-our-experiences/)
*   無料トライアルシステム
*   SAP 開発者センター
*   [トレーニング教材](https://noeldcosta.com/ja/sap-solution-builder/)
*   イベント登録

覚えておいていただきたいのは、Pユーザーはライセンスサポートやダウンロードへのアクセスを提供しないということです。それはSユーザーの特権です。ただし、Pユーザーは転職しても引き続きご利用いただけます。

私はコミュニティの質問、ブログのチェック、開発者リソースへのアクセスに自分のアカウントを頻繁に使用しています。これは基本的に、SAP エコシステムにおける個人の ID です。

後でユニバーサル ID を設定するときに、この P ユーザーをそれにリンクして、シングル サインオン アクセスを実現します。

## リンクされたアカウントの管理

複数の SAP アカウントがユニバーサル ID にリンクされています。これらを適切に管理することで、多くの手間が省けます。

*   **リンクされたアカウント間の切り替えは非常に簡単です****.** 
    *   ユニバーサルIDでログイン後、右上のプロフィール写真をクリックします。リンクされているすべてのアカウントがドロップダウンに表示されます。
    *   使いたいものをクリックするだけです。システムがコンテキストを切り替えるので、再度ログインする必要はありません。私はクライアント環境間を移動する時に、この機能を頻繁に使用しています。
*   **さまざまな SAP サービスに正しい ID を使用することは重要です。一部のサービスは特定のアカウント タイプでのみ機能します。** サポート チケットとダウンロードには、適切な権限を持つ S ユーザーが必要です。コミュニティの投稿には P ユーザーの ID が使用されます。間違った ID タイプを使用すると、アクセス エラーが発生してイライラすることになります。
*   **Sユーザーの権限を管理するには、会社の [SAP](https://noeldcosta.com/ja/best-sap-implementation-templates-activate-2024/)** **.** 管理者は、SAP サポートポータルのユーザー管理ワークスペースを通じてアクセスできるものを制御します。追加のアクセス権 (ライセンス管理やシステム登録など) が必要な場合は、管理者にリクエストする必要があります。ユニバーサル ID を通じて変更することはできません。

ユニバーサル ID はシングル サインオンの利便性を提供しますが、各アカウントの基本的な権限は変更されないことに注意してください。リンクされた各アカウントには、それぞれ個別のアクセス権が保持されます。

## よくある問題と解決策

私は長年にわたり、SAP ID の問題に何度も遭遇してきました。最も一般的な問題の解決策は次のとおりです。

### 1\. SAP Admin が見つからない – ヘルプをリクエストする方法

SAP管理者が誰なのか分からない場合は、IT部門に問い合わせてください。通常、IT部門は誰がSAP管理者なのかを知っています。 [SAPを管理する](https://noeldcosta.com/ja/2024-sap-timeline-planning-implementation-guide-essentials/) アクセスしてください。小規模な企業では、 [SAPプロジェクトマネージャー](https://noeldcosta.com/ja/how-to-create-an-sap-implementation-project-charter/) または IT ディレクター。それでも問題が解決しない場合は、財務チームに確認してください。財務チームは通常、ライセンス上の理由から S ユーザー アクセス権を持っており、管理者が誰であるかを知っています。

最後の手段として、SAP の一般サポート チャネルを通じて直接 SAP に連絡してください。SAP は貴社のスーパー管理者の連絡先情報を調べることはできますが、直接アクセスを許可することはできません。

### 2\. リンクエラー – 資格情報の不一致、アカウントのロック

アカウントのリンクに失敗した場合は、まず認証情報を再確認してください。ほとんどの失敗の原因は入力ミスです。ユーザー名とパスワードが正しいと確信していても、何度も失敗するとアカウントがロックされる可能性があります。これは少なくとも年に 1 回は発生します。

ロックされたアカウントについては、S ユーザーの場合は会社の SAP 管理者に連絡するか、P ユーザーの場合はパスワード リセット機能を使用する必要があります。リセット後、再度リンクを試みる前に約 15 分待ってください。システムは即座に同期されません。

同じメール アドレスを持つアカウントが問題を引き起こす場合があります。同じメール アドレスを使用して複数のアカウントを作成した場合は、SAP サポートに連絡して問題を解決する必要があります。

### 3\. メールが認証されていない – トラブルシューティング

確認メールは紛失したり、迷惑メールフォルダに入ってしまうことがあります。まずは迷惑メールを確認してください。10 分経ってもメールが届かない場合は、「確認メールを再送信」オプションを使用してください。

登録時に間違ったメールアドレスを入力した場合は、正しいアドレスで再度登録する必要があります。未確認のメールアドレスを変更することはできません。

確認リンクが機能しない場合は、クリックするのではなく、URL をブラウザに直接コピーしてみてください。一部のメール クライアントでは、リンクのフォーマットが壊れることがあります。

## SAP ユニバーサル ID: 一般的なエラーと修正 (完全なリスト)

SAP Universal ID を使ってアカウントをリンクしようとした際に、すぐには意味が分からないエラーが発生したことがあるかもしれません。システムによって S ユーザーがブロックされたり、すべて正しく表示されているにもかかわらずメールアドレスが一致しないという警告が表示されたりしたかもしれません。 

SAP Universal IDはアクセスを簡素化することを目的としています。しかし、リンク設定は予想以上に難しいと感じることがよくあります。メールアドレスが未確認という些細な理由でアカウントの接続に失敗するケースも見てきました。こうしたエラーは頻繁に発生し、修正には通常、いくつかの手動手順が必要です。 

以下に、最も一般的な問題とその解決方法の詳細を示します。

### 1\. 「このユーザーIDはすでに別のSAPユニバーサルIDにリンクされています」

*   **原因となる**: S ユーザーまたは P ユーザーは既に別の SAP Universal ID に関連付けられています。

*   **解像度**:

    *   すでにリンクされているユニバーサル ID にログインします。

    *   MFAデバイスに移動する **アカウント設定** > **リンクされたアカウント**.

    *   そこからアカウントのリンクを解除します。

    *   目的のユニバーサル ID を使用してリンクを再試行します。

    *   元のユニバーサルIDにアクセスできない場合は、コンポーネントでチケットを発行してください。 `XX-SER-SAPSMP-USR`.

### 2\. 「このユーザーIDをリンクする権限がありません」

*   **原因となる**: 管理権限のない ID をリンクしようとしています (会社が間違っているか、権限がありません)。

*   **解像度**:

    *   S ユーザー/P ユーザーの正当な所有者またはユーザーであることを確認します。

    *   会社の SAP 管理者にリンクまたはアクセスの許可を依頼してください。

    *   パートナー/顧客ユーザーをリンクする場合は、そのユーザーがパートナー ネットワークの範囲内にいることを確認してください。

### 3\. 「メールアドレスが一致しません - ユーザーIDをリンクできません」

*   **原因となる**: S ユーザー/P ユーザーのメール アドレスがユニバーサル ID のメール アドレスと一致しません。

*   **解像度**:

    *   アカウント間で電子メール アドレスを揃えます。

        *   ユニバーサルIDのメールアドレスを更新するには [アカウント](https://accounts.sap.com).

        *   SAP管理者にSユーザーのメールを更新するよう依頼してください。 **ユーザー管理プラットフォーム** サポート ポータルで。

### 4\. 「このSユーザーは非アクティブまたは削除されています」

*   **原因となる**: S ユーザーは非アクティブ化または期限切れになっています。

*   **解像度**:

    *   社内の SAP 管理者に問い合わせてください。

    *   再アクティブ化または新しい S ユーザーの発行をリクエストします。

    *   管理者は以下からこれを行うことができます **サポートポータル** > **ユーザー管理プラットフォーム**.

### 5\. 「技術的なエラーです。しばらくしてからもう一度お試しください」

*   **原因となる**: リンク中にバックエンドの問題またはセッションの期限が切れました。

*   **解像度**:

    *   すべての SAP プラットフォームからログアウトします。

    *   ブラウザのキャッシュ/Cookieをクリアします。

    *   シークレット/プライベートモードで再試行してください。

    *   それでも解決しない場合は、SAP にサポート チケットを開いてください。

### 6\. 「ログインするにはユニバーサルIDを使用する必要があります」

*   **原因となる**: リンク後に古い S ユーザー/P ユーザーの資格情報を使用しようとしています。

*   **解像度**:

    *   常にログインするには **ユニバーサルIDのメールアドレスとパスワード** at [ログイン.sap.com](https://login.sap.com).

    *   一度リンクされた S ユーザーの資格情報を直接使用することは避けてください。

### 7\. 「試行回数が多すぎます。アカウントを一時的にロックしました。」

*   **原因となる**: ログインまたはリンクの試行が複数回失敗したため、ロックアウトが発生しました。

*   **解像度**:

    *   自動ロック解除まで15〜30分お待ちください。

    *   または、以下の方法でパスワードをリセットしてください。 accounts.sap.com > パスワードを忘れた場合.

### 8\. 「このSユーザーには有効な役割が割り当てられていません」

*   **原因となる**: S ユーザーは存在しますが、割り当てられた権限がありません (例: サポート アクセス)。

*   **解像度**:

    *   SAP 管理者に問い合わせてください。

    *   必要な役割を割り当てるには **ユーザー管理プラットフォーム** サポート ポータルで。

    *   最低限推奨: インシデントの表示、プロファイルの編集、ソフトウェアのダウンロード。

### 9\. 「SAP Universal ID に XNUMX 番目のメールリンクがありません。」

*   **原因となる**: 予備のメールアドレスを追加しましたが、確認していません。

*   **解像度**:

    *   ログイン時に [アカウント](https://accounts.sap.com).

    *   に行く **個人情報** > **メールアドレス**.

    *   確認メールを再送信します。

    *   確認メール内のリンクをクリックして、プロセスを完了します。

### 10\. 「セッションの有効期限が切れました」/「無効なトークン」

*   **原因となる**: セッションがタイムアウトしたか、認証トークンが古くなっています。

*   **解像度**:

    *   すべての SAP システムからログアウトします。

    *   ブラウザのキャッシュをクリアします。

    *   最初から再度ログインし、リンクフローを繰り返します。

### 11\. 「異なる地域/市場のユーザーIDをリンクしようとしています」

*   **原因となる**: 別の SAP エンティティまたはリージョンからのユーザー ID をリンクしようとしています。

*   **解像度**:

    *   リンクを手動で承認するには、SAP にサポート リクエストを送信してください。

    *   地域アカウントの有効性を確認します。

    *   これは、プライマリ地域外から SAP Business Network または Ariba ID をリンクするときに発生することがあります。

### 12\. 「リンクしようとしているユーザーIDの形式が正しくありません」

*   **原因となる**: 無効な ID (ベンダー ID、SCN ID、顧客番号など) を入力しています。

*   **解像度**:

    *   リンクのみ有効:

        *   **Sユーザー** （例えば、 `S0001234567`)

        *   **Pユーザー** （例えば、 `P12345678`)

    *   ビジネス パートナー番号や古い SAP ID を入力しないでください。

### 13\. 「アカウントリンクの制限に達しました」

*   **原因となる**: SAP Universal ID には、リンクされたアカウントの制限 (通常は 10) があります。

*   **解像度**:

    *   に行く **リンクされたアカウント** at [アカウント](https://accounts.sap.com).

    *   不要になったアカウントのリンクを解除します。

    *   次に、新しいユーザー ID のリンクに進みます。

### 14\. SAPユニバーサルIDからSユーザーまたはPユーザーのリンクを解除する方法

**ステップ：**

1.  に行く [アカウント](https://accounts.sap.com)

2.  ユニバーサルIDでログインしてください。

3.  詳しくはこちら **アカウント設定** > **リンクされたアカウント**.

4.  削除する S ユーザーまたは P ユーザーを見つけます。

5.  詳しくはこちら **リンク解除/削除**.

6.  確認してください。

**重要：**  
リンクを解除できない場合（例：ユニバーサルIDへのアクセスが失われた場合）は、コンポーネントのサポートチケットを発行してください。 `XX-SER-SAPSMP-USR`.

## 結論

私はSAPユニバーサルIDが初めて登場した時から使っています。 [すべてのSAPを管理](https://noeldcosta.com/ja/sap-enterprise-warehouse-management-sap-ewm-essentials/) ログインがずっと簡単になりました。1つのログインですべてに対応できるので、パスワードのやりとりは不要、システム間の切り替えも簡単です。

『Brooklyn Galaxy』のために、倪氏はブルックリン美術館のコレクションからXNUMX点の名品を選び、そのイメージを極めて詳細に描き込みました。これらの作品は、彼の作品とともに中国ギャラリーに展示されています。彼はXNUMX年にこの作品の制作を開始しましたが、最初の硬貨には、当館が所蔵する [顧客間を行き来するコンサルタントや、異なるSAPを使用している人](https://noeldcosta.com/ja/best-sap-technical-change-management-tools-2025/) サービスでは、ユニバーサル ID によって膨大な時間が節約されます。パスワードを探したり、忘れたアカウントをリセットしたりする手間が省けるので、おそらく毎日 10 ～ 15 分は節約できます。

ただし、S ユーザーの管理は依然として重要です。これらの企業アカウントは、ライセンスされたアイテムとサポートへのアクセスを制御します。企業は SAP に多額の費用を支払っているため、誰がアクセスできるかを制御します。

ヘルプが必要な場合は、まず会社の SAP 管理者に相談してください。彼らは S ユーザーの問題のほとんどを処理しており、会社の設定を知っています。ユニバーサル ID の技術的な問題については、SAP ヘルプ ポータルに適切なドキュメントとサポートがあります。

設定には 10 分ほどかかりますが、数え切れないほどの時間を節約できます。SAP を定期的に使用する場合は、ぜひ設定してください。

ユニバーサルIDはシステム変更や新規導入の際に真価を発揮します。 [SAP ロールアウト](https://noeldcosta.com/ja/creating-an-effective-sap-project-steering-committee/)すべてに新しいログインを行う代わりに、ユニバーサル ID にリンクするだけです。誰にとっても簡単になります。

ただし、ユニバーサル ID はログインに役立ちますが、権限は変更されないことに注意してください。各アカウントは特定のアクセス権を保持します。

経験から言うと、 [SAP](https://noeldcosta.com/ja/sap-ecc-to-s4hana-migration-case-study/) IDを整理することで大きな成果が得られます。SAPを週に1回使用する場合でも、毎日一日中使用する場合でも、すべてがスムーズになります。

## ご質問があるかもしれません...

### 1\. SAP Universal ID とは何ですか? また、なぜ使用する必要があるのですか?

SAP ユニバーサル ID は、すべての SAP アカウントを接続する単一のログインです。これにより、さまざまなパスワードやユーザー名を操作することなく、複数の SAP システムにアクセスできます。

### 2\. P ユーザー アカウントと S ユーザー アカウントの両方を SAP Universal ID にリンクできますか?

はい、個人の P ユーザー アカウントと会社の S ユーザー アカウントの両方をユニバーサル ID にリンクできます。ログアウトして再度ログインしなくても、アカウント間の切り替えが非常に簡単になります。

### 3\. SAP Universal ID を作成すると、会社に通知されますか?

ユニバーサル ID の作成は個人で行うもので、会社とは別のものです。ただし、会社の S ユーザーをリンクすると、SAP 管理者は引き続きそれらのアクティビティを確認できます。

### 4\. 会社を退職した場合、SAP Universal ID は失われますか?

いいえ、SAP ユニバーサル ID はそのまま残ります。会社の S ユーザーには確実にアクセスできなくなりますが、ユニバーサル ID と P ユーザーは引き続きお客様のものとなります。

### 5\. SAP Universal ID の電子メール アドレスを変更できますか?

はい、アカウント設定に移動して更新するだけです。変更後は新しいメールを確認することを忘れないでください。確認しないとロックアウトされます。

### 6\. SAP Universal ID にリンクできるアカウントの数はいくつですか?

実際の制限はありません。私は、さまざまなクライアントからの複数の S ユーザーと、数人の P ユーザーをすべて自分のものにリンクしています。

### 7\. リンクされたアカウントのパスワードの有効期限が切れるとどうなりますか?

パスワードの有効期限が切れた S ユーザーの場合は、会社の SAP 管理者に連絡する必要があります。P ユーザーの場合は、リセット機能を使用するだけです。

### 8\. SAP Universal ID を使用すると、SAP システムでより多くの権限が付与されますか?

まったくそうではありません。ログインが簡単になるだけです。実際の権限は各アカウントによって異なります。

### 9\. SAP Universal ID からアカウントのリンクを解除するにはどうすればよいですか?

「アカウント設定」から「リンクされたアカウント」に移動し、削除したいアカウントを見つけて「リンク解除」をクリックします。とても簡単です。

### 10\. SAP システムにアクセスするには SAP Universal ID が必要ですか?

これはオプションですが、やる価値は十分にあります。必要に応じて個別の認証情報を使用することもできますが、なぜ面倒なことになるのでしょうか?
