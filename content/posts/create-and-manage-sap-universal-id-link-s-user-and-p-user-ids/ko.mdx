# SAP 유니버설 ID 생성 및 관리: S-User 및 P-User ID 연결


### 노엘 디코스타

*   마지막 업데이트 : 2025 년 3 월 16 일

SAP를 처음 접했을 때, SAP 유니버설 ID, S-User 역할, P-User 역할의 개념과 이들이 어떻게 상호 작용하는지 이해하려고 몇 주 동안 애썼습니다. SAP 유니버설 ID가 도입되기 전에는 모든 계정을 관리하는 것이 정말 악몽 같았습니다.

SAP Universal ID는 컨설턴트와 고객 모두에게 골칫거리였던 문제를 해결해 줍니다. 이를 통해 다음과 같은 작업을 수행할 수 있습니다. [여러 SAP 관리](https://noeldcosta.com/ko/start-your-sap-implementation-project-right/) 하나의 로그인으로 여러 계정을 관리하세요. 더 이상 비밀번호를 스프레드시트에 입력하거나, 어떤 계정 정보를 어디에 입력해야 하는지 잊어버렸을 때 끊임없이 비밀번호 재설정 이메일을 확인할 필요가 없습니다.

여러 고객 시스템을 오가며 작업하는 컨설턴트에게 이 기능은 엄청난 시간 절약 효과를 가져다줍니다. 매번 로그아웃하고 다시 로그인할 필요 없이 P-User 계정 간에 빠르게 전환할 수 있습니다. 여러 시스템 환경에서 작업할 때 이러한 시간 절약 효과는 빠르게 누적됩니다.

S-User 계정(관리자 계정) [SAP 지원](https://noeldcosta.com/ko/best-sap-implementation-strategies-to-avoid-costly-mistakes/) 라이선스 및 라이선스 관리도 유니버설 ID에 연결됩니다. 지원 티켓을 제출하거나 소프트웨어를 다운로드할 때 훨씬 편리합니다.

아직 구현이 완벽하지 않습니다. 일부 구형 모델 [SAP 시스템은 통합되지 않습니다.](https://noeldcosta.com/ko/sap-implementation-cost-and-budget-breakdown/) 원활하게 작동합니다. 하지만 기존 계정이 많은 경우 초기 설정이 다소 혼란스러울 수 있습니다.

하지만 전반적으로 유니버설은 [ID는 일상적인 불편함 중 하나를 해결해 줍니다.](https://noeldcosta.com/ko/5-best-crm-systems-for-sap-in-2024/) SAP 생태계에서 사용 가능합니다. 여러 클라이언트나 시스템을 관리하는 경우, 이 설정을 할 일 목록에 추가하는 것이 좋습니다. 설정하는 데 약 15분 정도 소요되며, 향후 로그인 관련 문제로 발생하는 수많은 시간을 절약해 줍니다.

SAP Universal ID는 여러분의 업무를 간소화합니다. [SAP 경험](https://noeldcosta.com/ko/sap-job-description-generator/) 모든 계정을 하나의 로그인으로 연결하세요. 여러 계정의 로그인 정보를 기억해야 하는 번거로움을 없애고, 연결된 각 계정의 특정 권한을 유지하면서 시간과 불편함을 줄일 수 있습니다.

## SAP 사용자 유형 이해하기

저는 수년간 다양한 SAP ID 유형을 다뤄왔습니다. 각 ID는 고유한 목적을 가지고 있으며, 올바른 접근 권한을 얻으려면 그 차이점을 아는 것이 중요합니다.

**P-사용자 ID** 이 여러분의 [개인용 SAP](https://noeldcosta.com/ko/ai-insights-shiftgearx-noeldcosta/) SAP ID는 회사와 상관없이 개인에게 영구적으로 부여됩니다. SAP 커뮤니티에 가입하거나, 평가판을 다운로드하거나, 포럼에 글을 올릴 때 사용하죠. 저는 여러 직장을 거치는 동안 16년(정확히는 기억 안 나지만) 동안 이 ID를 계속 사용해 왔습니다. 말하자면 개인 SAP 여권과 같은 역할을 하는 거죠.

**S-사용자 ID** 작동 방식이 완전히 다릅니다. 이것은 당신이 아닌 회사 소유입니다. 이 ID를 통해 라이선스를 취득할 수 있습니다. [SAP 서비스](https://noeldcosta.com/ko/best-sap-documentation-tools-2024-guide/)회사에서 비용을 지불하는 지원 티켓 및 다운로드 횟수 등이 포함됩니다. 회사를 떠나면 S-User 계정을 잃게 됩니다. 저는 회사를 옮기면서 여섯 번이나 다른 S-User 계정을 받았습니다.

[SAP Universal ID 문제가 드디어 해결되었습니다.](https://noeldcosta.com/ko/why-sap-data-migration-fails-and-how-to-fix-it/) 이 복잡한 시스템을 없애줍니다. 하나의 로그인으로 여러 S-User를 P-User와 연결할 수 있습니다. 비밀번호도 5개, 6개가 아닌 하나만 있으면 됩니다. 회사를 옮기더라도 새 S-User를 유니버설 ID에 연결하기만 하면 됩니다. 같은 주에 여러 클라이언트 시스템을 사용해야 할 때 정말 유용했습니다.

## SAP 유니버설 ID 생성 단계별 가이드

## \# 1단계: 유니버설 ID 설정

SAP 유니버설 ID를 설정해야 하는 경우, 설정 방법은 다음과 같습니다.

*   SAP Universal ID 사이트로 이동하세요. [account.sap.com](https://account.sap.com)그러면 SAP 메인 로그인 페이지로 이동합니다. 모든 것이 여기서 시작됩니다.


*   **"레지스터"** 아주 잘 보여서 놓칠 수가 없어요. 이미 어떤 종류의 물건을 가지고 계시다면... [SAP 계정](https://noeldcosta.com/ko/sap-quality-gates-implementation/)다른 옵션이 보일 수도 있지만, 이 버튼은 여전히 남아 있어야 합니다.


*   개인 정보를 입력하세요. 다음 정보를 제공해야 합니다.  
    *   이름과 성
    *   이메일 주소 (임시로 발급받은 회사 이메일 주소가 아닌, 장기적으로 사용할 이메일 주소를 사용하세요)
    *   강력한 비밀번호를 만드세요 (최소 8자 이상, 대문자, 소문자, 숫자 및 특수 문자를 포함해야 합니다).
    *   귀하의 거주 국가
    *   회사명 (선택 사항이지만 도움이 됩니다)
*   수락 [조건](https://noeldcosta.com/ko/sap-negotiation-advisors-reduce-cost/) 이용 약관에는 SAP의 이용 약관에 동의하는 체크박스가 있습니다. 진행하려면 이 체크박스를 클릭해야 합니다. 마케팅 동의를 위한 선택 체크박스도 있습니다. [연락](https://noeldcosta.com/ko/2025-the-year-sap-generative-ai-redefines-middle-east-careers/)그건 당신이 결정하세요.
*   이메일 주소를 인증하세요. SAP에서 입력하신 이메일 주소로 인증 코드를 보내드립니다. 받은 편지함(필요한 경우 스팸 폴더 포함)을 확인하세요. 10분 이내에 인증 페이지에 해당 코드를 입력하세요. 10분 이내에 코드를 입력하지 않으면 새 코드를 요청해야 합니다. 회사 이메일 주소가 아닌 개인 이메일 주소를 사용하세요.
*   새로운 유니버설 ID로 로그인하세요. 인증이 완료되면 새 계정 정보로 로그인하라는 메시지가 표시됩니다. 이제 유니버설 ID가 활성화되었습니다.
*   (링크) [기존 SAP](https://noeldcosta.com/ko/mastering-sap-implementation-a-step-by-step-guide-for-2025/) 로그인 후 기존 계정을 연결하는 옵션이 표시됩니다. P 사용자 또는 S 사용자 계정이 있는 경우 "계정 연결"을 클릭하고 안내에 따라 진행하세요. 연결하려는 각 계정의 사용자 이름과 비밀번호를 알고 있어야 합니다.

전체 과정은 약 10분 정도 소요됩니다. 모든 SAP 계정 정보에 접근할 수 있을 때 진행하는 것이 좋습니다. 그래야 한 번에 모든 것을 연결할 수 있습니다.

유니버설 ID는 직장 생활 내내 사용되므로 가능하면 회사 이메일보다는 개인 이메일을 사용하는 것이 좋습니다.

## \# 2단계: S-사용자 및 P-사용자 ID를 유니버설 ID에 연결

유니버설 ID를 설정하셨으니 이제 S-User 또는 P-User 계정에 연결해야 합니다. 어떻게 연결하나요?

*   SAP Universal ID로 로그인하세요. 다음으로 이동하세요. [account.sap.com](https://account.sap.com) 유니버설 ID 자격 증명으로 로그인하세요. **설정 과정에서 생성한 이메일 주소와 비밀번호를 사용하고 있는지 확인하십시오.**


*   로그인 후 "계정 설정" > "연결된 계정"으로 이동하세요. 오른쪽 상단에 있는 프로필 아이콘을 클릭하여 드롭다운 메뉴를 열고 "계정 설정"을 선택하세요. 왼쪽 사이드바에서 "연결된 계정" 옵션을 클릭하세요.
*   S 사용자 또는 P 사용자의 자격 증명을 입력하세요. 연결된 계정 페이지에는 현재 연결된 계정(있는 경우)과 추가 계정 추가 옵션이 표시됩니다. "계정 연결" 버튼을 클릭하세요. 연결할 계정의 사용자 이름과 비밀번호를 입력하라는 팝업 창이 나타납니다.
*   자격 증명을 입력한 후 "계정 연결"을 클릭하여 확인하고 연결하세요. [시스템이 정보를 검증합니다](https://noeldcosta.com/ko/my-journey-with-customer-information-solutions-defense/)성공적으로 연결되면 확인 메시지가 표시됩니다. 이제 해당 계정이 연결된 계정 목록에 나타납니다.
*   여러 계정에 대해 반복하세요. 연결하려는 계정마다 3~4단계를 반복할 수 있습니다. 유니버설 ID에 연결할 수 있는 계정 수에는 제한이 없습니다.

**저는 여러분이 모든 것을 갖추기를 권장합니다. [시작하기 전에 계정 자격 증명을 준비해 두세요.](https://noeldcosta.com/ko/5-best-crm-systems-for-sap-in-2024/) 이 프로세스를 진행하세요. 여러 SAP 고객사와 협업한 경우 이전 S-User ID를 잊어버리기 쉬우므로 연결한 계정을 기록해 두는 것이 좋습니다.**

계정을 연결하면 하나의 통합 ID로 로그인하여 모든 계정에 액세스할 수 있습니다. 시간을 크게 절약할 수 있습니다.

## S-User ID를 얻는 방법

S 사용자 ID는 회사에서 관리합니다. P 사용자 ID와 달리 개인이 직접 생성할 수 없습니다. 이러한 계정은 개인이 아닌 회사 소유입니다.

S-User를 얻으려면 다음을 수행하십시오. [SAP를 통해 요청하세요.](https://noeldcosta.com/ko/sap-project-scope-template-management-and-control/) 최고 관리자. 이 사람 [모든 S-User 계정을 관리합니다.](https://noeldcosta.com/ko/ai-governance-framework-guide-building-a-responsible-ai-plan/) 귀사에서 이 역할을 누가 맡고 있는지 모른다면 IT 부서에 문의하십시오. 소규모 회사에서는 보통 IT 이사 또는 [SAP 프로젝트 관리자](https://noeldcosta.com/ko/project-planning-and-control-get-sap-projects-back-on-track/).

럭셔리 [SAP 슈퍼 관리자 생성 과정](https://noeldcosta.com/ko/best-sap-implementation-templates-activate-2024/) 방법은 아주 간단합니다. 관리자 S-User 계정으로 SAP 지원 포털에 로그인한 후, "사용자 관리"로 이동하여 "S-User 생성"을 선택하면 됩니다. 

**이름, 이메일 주소, 전화번호가 필요합니다. 시스템에서 사용자 이름을 자동으로 생성합니다.**

역할 할당은 매우 중요합니다. 최고 관리자는 사용자가 접근할 수 있는 권한을 정확하게 지정합니다.

*   (티켓 생성 관련) 지원
*   라이선스(라이선스 관리용)
*   다운로드 (소프트웨어 다운로드용)
*   시스템 (시스템 등록용)

다리 [사용자는 지원이 필요합니다](https://noeldcosta.com/ko/change-management-plan-success/) 다운로드 역할도 있습니다. 라이선스 역할은 일반적으로 소수의 사람들에게만 부여됩니다.

계정이 생성되면 활성화 단계가 포함된 이메일을 받게 됩니다. 회사를 떠나면 S-User 계정은 회사 소유이므로 회사에서 자동으로 해지됩니다.


## P-User ID를 얻는 방법

저는 10년 넘게 P-User 계정을 사용해 왔습니다. S-User 계정과 달리 P-User ID는 고용주가 아닌 본인에게 속한 개인 계정입니다.

P-User 계정 생성은 간단합니다. SAP 커뮤니티(community.sap.com)에 접속하여 "등록" 버튼을 클릭하세요. 유효한 이메일 주소가 필요하며, 이 계정은 향후 경력 전반에 걸쳐 사용되므로 회사 이메일보다는 개인 이메일을 사용하는 것을 강력히 권장합니다.

등록 양식에는 이름, 거주지, 직업 정보 등의 기본 정보를 입력해야 합니다. 제출 후에는 확인 이메일을 받게 됩니다. 이메일에 있는 링크를 클릭하면 등록이 완료됩니다.

P-Users를 통해 다음 기능에 접근할 수 있습니다:

*   [SAP 커뮤니티 포럼 및 콘텐츠](https://noeldcosta.com/ko/write-for-us-lets-share-our-experiences/)
*   무료 체험 시스템
*   SAP 개발자 센터
*   [교육 자료](https://noeldcosta.com/ko/sap-solution-builder/)
*   이벤트 등록

가장 중요한 점은 P 사용자는 라이선스 지원이나 다운로드에 접근할 수 없다는 것입니다. 이는 S 사용자를 위한 기능입니다. 하지만 P 사용자는 직장을 옮기더라도 계정이 유지됩니다.

저는 커뮤니티 질문, 블로그 확인, 개발자 자료 접근 등 다양한 용도로 제 계정을 꾸준히 사용합니다. SAP 생태계에서 제 개인 식별 정보와 같은 역할을 하죠.

나중에 유니버설 ID를 설정할 때 이 P-User를 유니버설 ID에 연결하여 단일 로그인으로 액세스할 수 있게 됩니다.

## 연결된 계정 관리

저는 유니버설 ID에 여러 개의 SAP 계정을 연결했습니다. 이 계정들을 제대로 관리하면 골칫거리를 많이 줄일 수 있습니다.

*   **연동된 계정 간 전환은 매우 간단합니다.****.** 
    *   유니버설 ID로 로그인한 후, 오른쪽 상단에 있는 프로필 사진을 클릭하세요. 연결된 모든 계정이 표시되는 드롭다운 메뉴가 나타납니다.
    *   사용하고 싶은 항목을 클릭하기만 하면 됩니다. 시스템은 다시 로그인할 필요 없이 환경을 전환합니다. 저는 클라이언트 환경을 전환할 때 이 기능을 자주 사용합니다.
*   **SAP 서비스별로 올바른 ID를 사용하는 것은 매우 중요합니다. 일부 서비스는 특정 계정 유형에서만 작동합니다.** 지원 티켓을 제출하거나 파일을 다운로드하려면 올바른 권한을 가진 S-User 계정이 필요합니다. 커뮤니티 게시물을 작성할 때는 P-User 계정을 사용합니다. 잘못된 ID 유형을 사용하면 액세스 오류가 발생하여 불편을 겪을 수 있습니다.
*   **S-사용자에 대한 권한 관리는 회사 담당자와의 협력이 필요합니다. [SAP](https://noeldcosta.com/ko/best-sap-implementation-templates-activate-2024/) 관리자****.** 관리자는 SAP 지원 포털의 사용자 관리 작업 공간을 통해 사용자가 접근할 수 있는 권한을 제어합니다. 라이선스 관리 또는 시스템 등록과 같은 추가 접근 권한이 필요한 경우 관리자에게 요청해야 합니다. 이러한 권한은 유니버설 ID를 통해 변경할 수 없습니다.

유니버설 ID는 단일 로그인으로 편리함을 제공하지만, 각 계정의 기본 권한은 변경되지 않는다는 점을 기억하세요. 연결된 각 계정은 여전히 고유한 접근 권한을 가지고 있습니다.

## 일반적인 문제 및 해결 방법

저는 수년간 SAP ID 관련 문제를 많이 겪어왔습니다. 가장 흔한 문제들에 대한 해결 방법을 알려드리겠습니다.

### 1\. SAP 관리자를 찾을 수 없습니다 - 도움 요청 방법

SAP 관리자가 누구인지 알 수 없다면 IT 부서에 문의하세요. IT 부서에서는 일반적으로 SAP 관리자가 누구인지 알고 있습니다. [SAP를 관리합니다](https://noeldcosta.com/ko/2024-sap-timeline-planning-implementation-guide-essentials/) 접근 권한. 소규모 회사에서는 직원에게 직접 물어보는 것을 시도해 보세요. [SAP 프로젝트 관리자](https://noeldcosta.com/ko/how-to-create-an-sap-implementation-project-charter/) 또는 IT 부서장에게 문의해 보세요. 그래도 해결되지 않으면 재무팀에 문의해 보세요. 재무팀은 라이선스 문제로 S-User 접근 권한을 가지고 있는 경우가 많으며, 관리자가 누구인지 알고 있을 것입니다.

최후의 수단으로 SAP의 일반 지원 채널을 통해 직접 문의하십시오. SAP 담당자가 귀사의 최고 관리자 연락처 정보를 조회해 줄 수는 있지만, 직접 접근 권한을 부여해 주지는 않습니다.

### 2\. 연결 오류 – 자격 증명 불일치, 계정 잠금

계정 연결에 실패하면 먼저 로그인 정보를 다시 확인하세요. 대부분의 실패 원인은 오타입니다. 사용자 이름과 비밀번호가 정확하다면, 로그인 시도 횟수가 너무 많아 계정이 잠겼을 수도 있습니다. 저도 매년 최소 한 번은 이런 일을 겪습니다.

잠긴 계정의 경우, S 사용자는 회사 SAP 관리자에게 문의하고, P 사용자는 비밀번호 재설정 기능을 사용해야 합니다. 재설정 후 약 15분 정도 기다린 후 다시 연결을 시도하십시오. 시스템 동기화는 즉시 이루어지지 않습니다.

동일한 이메일 주소를 사용하는 계정은 때때로 문제를 일으킬 수 있습니다. 동일한 이메일 주소로 여러 계정을 생성한 경우, SAP 지원팀에 문의하여 문제를 해결해야 합니다.

### 3\. 이메일 인증 실패 - 문제 해결

인증 이메일이 간혹 누락되거나 스팸 폴더로 이동하는 경우가 있습니다. 먼저 스팸 메일함을 확인해 보세요. 10분 후에도 이메일을 받지 못했다면 "인증 이메일 재전송" 옵션을 사용해 주세요.

회원가입 시 잘못된 이메일 주소를 입력했다면, 올바른 주소로 다시 회원가입을 해야 합니다. 인증되지 않은 이메일 주소는 변경할 수 없습니다.

링크가 작동하지 않는 경우, 링크를 클릭하는 대신 URL을 직접 복사하여 브라우저에 붙여넣어 보세요. 일부 이메일 클라이언트에서 링크 형식이 깨지는 경우가 있습니다.

## SAP Universal ID: 일반적인 오류 및 해결 방법(전체 목록)

SAP Universal ID를 통해 계정을 연결하려고 시도한 경우, 즉시 이해하기 어려운 오류가 발생했을 수 있습니다. 시스템에서 S-User를 차단했거나, 모든 것이 올바른 것처럼 보였는데도 이메일 주소 불일치 경고가 표시되었을 수 있습니다. 

SAP Universal ID는 접근을 간소화하기 위해 만들어졌습니다. 하지만 연결 과정은 예상보다 복잡하게 느껴지는 경우가 많습니다. 이메일 주소가 인증되지 않은 것과 같은 사소한 이유로 계정 연결에 실패하는 경우도 종종 발생합니다. 이러한 오류는 자주 발생하며, 해결하려면 몇 가지 수동 작업을 거쳐야 합니다. 

아래에서는 가장 흔한 문제점과 해결 방법을 자세히 설명합니다.

### 1\. "이 사용자 ID는 이미 다른 SAP Universal ID에 연결되어 있습니다."

*   **원인**S 사용자 또는 P 사용자는 이미 다른 SAP 유니버설 ID에 연결되어 있습니다.

*   **분해능**:

    *   이미 연결된 유니버설 ID로 로그인하세요.

    *   로 이동 **계정 설정** > **연결된 계정**.

    *   거기서 계정 연결을 해제하세요.

    *   원하는 유니버설 ID로 연결을 다시 시도하십시오.

    *   원래 유니버설 ID에 접근할 수 없는 경우, 구성 요소 항목 아래에 티켓을 제출하십시오. `XX-SER-SAPSMP-USR`.

### 2\. "이 사용자 ID를 연결할 권한이 없습니다."

*   **원인**권한이 없는 ID(잘못된 회사 또는 권한 부족)를 연결하려고 시도하고 있습니다.

*   **분해능**:

    *   본인이 S-사용자/P-사용자의 정당한 소유자 또는 사용자임을 확인하십시오.

    *   회사 SAP 관리자에게 연결 또는 액세스 권한 부여를 요청하십시오.

    *   파트너/고객 사용자를 연결하는 경우 해당 사용자가 파트너 네트워크 범위 내에 있는지 확인하십시오.

### 3\. "이메일 주소 불일치 - 사용자 ID를 연결할 수 없습니다"

*   **원인**S 사용자/P 사용자의 이메일 주소가 유니버설 ID 이메일 주소와 일치하지 않습니다.

*   **분해능**:

    *   모든 계정의 이메일 주소를 일관되게 사용하세요.

        *   유니버설 ID 이메일을 업데이트하려면 다음 링크를 클릭하세요. [accounts.sap.com](https://accounts.sap.com).

        *   SAP 관리자에게 S-사용자의 이메일 주소를 업데이트해달라고 요청하세요. **사용자 관리** 지원 포털에서 확인하세요.

### 4\. "이 S 사용자는 비활성화되었거나 삭제되었습니다."

*   **원인**S 사용자가 비활성화되었거나 만료되었습니다.

*   **분해능**:

    *   내부 SAP 관리자에게 문의하십시오.

    *   S 사용자 계정 재활성화 또는 신규 발급을 요청합니다.

    *   관리자는 다음을 통해 이를 수행할 수 있습니다. **지원 포털** > **사용자 관리**.

### 5\. "기술적 오류가 발생했습니다. 나중에 다시 시도해 주세요."

*   **원인**연결 과정 중 백엔드 문제 또는 세션 만료가 발생했습니다.

*   **분해능**:

    *   모든 SAP 플랫폼에서 로그아웃하십시오.

    *   브라우저 캐시/쿠키를 삭제하세요.

    *   시크릿/개인 모드에서 다시 시도하세요.

    *   문제가 지속되면 SAP에 지원 티켓을 제출하십시오.

### 6\. "로그인하려면 유니버설 ID를 사용해야 합니다."

*   **원인**연결 후 이전 S 사용자/P 사용자 자격 증명을 사용하려고 시도했습니다.

*   **분해능**:

    *   항상 다음 방법을 통해 로그인하세요. **유니버설 ID 이메일 및 비밀번호** at [login.sap.com](https://login.sap.com).

    *   일단 연결되면 S-user 자격 증명을 직접 사용하지 마십시오.

### 7\. "시도 횟수 초과. 계정이 일시적으로 잠겼습니다."

*   **원인**로그인 또는 연결 시도 실패가 여러 번 발생하면 계정이 잠깁니다.

*   **분해능**:

    *   자동 잠금 해제까지 15~30분 정도 기다려 주세요.

    *   또는 다음을 통해 비밀번호를 재설정하세요. accounts.sap.com > 비밀번호를 잊으셨나요?.

### 8\. "이 S 사용자에게 할당된 유효한 역할이 없습니다."

*   **원인**S 사용자는 존재하지만 할당된 권한(예: 지원 접근 권한)이 없습니다.

*   **분해능**:

    *   SAP 관리자에게 문의하십시오.

    *   필요한 역할을 할당하려면 다음을 사용하세요. **사용자 관리** 지원 포털에서 확인하세요.

    *   최소 권장 기능: 사건 조회, 프로필 편집, 소프트웨어 다운로드.

### 9\. "SAP Universal ID에 두 번째 이메일 링크가 누락되었습니다!"

*   **원인**보조 이메일을 추가하셨지만 아직 인증하지 않으셨습니다.

*   **분해능**:

    *   로그인 [accounts.sap.com](https://accounts.sap.com).

    *   We Buy Orders 신청서를 클릭하세요. **개인 정보** > **이메일 주소**.

    *   인증 이메일을 다시 보내주세요.

    *   인증 이메일에 있는 링크를 클릭하여 절차를 완료하세요.

### 10\. "세션이 만료되었습니다" / "유효하지 않은 토큰입니다"

*   **원인**세션 시간 초과 또는 인증 토큰 만료.

*   **분해능**:

    *   모든 SAP 시스템에서 로그아웃하십시오.

    *   브라우저 캐시를 지 웁니다.

    *   완전히 새로 로그인한 후 연결 과정을 다시 진행하세요.

### 11\. "다른 지역/시장의 사용자 ID를 연결하려고 합니다."

*   **원인**다른 SAP 엔티티 또는 영역의 사용자 ID를 연결하려고 시도하고 있습니다.

*   **분해능**:

    *   SAP에 지원 요청을 제출하여 링크를 수동으로 승인받으세요.

    *   지역 계정의 유효성을 확인하십시오.

    *   이는 기본 지역 외에서 SAP Business Network 또는 Ariba ID를 연결할 때 가끔 발생하는 현상입니다.

### 12\. "연결하려는 사용자 ID 형식이 올바르지 않습니다."

*   **원인**잘못된 ID를 입력했습니다(예: 공급업체 ID, SCN ID, 고객 번호).

*   **분해능**:

    *   유효한 링크는 다음과 같습니다.

        *   **S 사용자** (예 : `S0001234567`)

        *   **P-유저** (예 : `P12345678`)

    *   비즈니스 파트너 번호 또는 더 이상 사용되지 않는 SAP ID를 입력하지 마십시오.

### 13\. "계정 연결 한도에 도달했습니다"

*   **원인**SAP Universal ID는 연결된 계정 수에 제한(일반적으로 10개)이 있습니다.

*   **분해능**:

    *   We Buy Orders 신청서를 클릭하세요. **연결된 계정** at [accounts.sap.com](https://accounts.sap.com).

    *   계정 연결 해제는 더 이상 필요하지 않습니다.

    *   다음으로 새 사용자 ID를 연결하십시오.

### 14\. SAP Universal ID에서 S-User 또는 P-User 연결을 해제하는 방법

**단계 :**

1.  We Buy Orders 신청서를 클릭하세요. [accounts.sap.com](https://accounts.sap.com)

2.  유니버설 ID로 로그인하세요.

3.  **계정 설정** > **연결된 계정**.

4.  삭제하려는 S 사용자 또는 P 사용자를 찾으십시오.

5.  **연결 해제/삭제**.

6.  확인.

**중요 사항:**  
연결 해제가 불가능한 경우(예: 유니버설 ID 액세스 권한 상실) 구성 요소 항목에 지원 티켓을 제출하십시오. `XX-SER-SAPSMP-USR`.

## 맺음말

저는 SAP Universal ID가 처음 출시되었을 때부터 사용해 왔습니다. [내 모든 SAP 관리하기](https://noeldcosta.com/ko/sap-enterprise-warehouse-management-sap-ewm-essentials/) 로그인 방식이 훨씬 간편해졌습니다. 모든 것에 하나의 로그인으로 접속할 수 있어 더 이상 비밀번호를 신경 쓸 필요가 없고, 시스템 간 전환도 쉽습니다.

럭셔리 [여러 고객사를 옮겨 다니는 컨설턴트나 서로 다른 SAP 시스템을 사용하는 사람](https://noeldcosta.com/ko/best-sap-technical-change-management-tools-2025/) Universal ID 덕분에 여러 서비스를 이용하면서 시간을 엄청나게 절약할 수 있었습니다. 비밀번호를 찾거나 잊어버린 계정을 재설정하는 데 드는 시간을 줄여주니 매일 10~15분 정도는 아낄 수 있는 것 같습니다.

하지만 S-User 관리는 여전히 중요합니다. 이러한 회사 계정은 라이선스가 부여된 항목과 지원에 대한 접근 권한을 제어합니다. 기업은 SAP에 막대한 비용을 투자하기 때문에 접근 권한을 통제할 권리가 있습니다.

도움이 필요할 때는 먼저 회사 SAP 관리자에게 문의하세요. SAP 관리자는 대부분의 S-User 관련 문제를 처리하며 회사 시스템 구성에 대해 잘 알고 있습니다. 유니버설 ID 관련 기술적인 문제는 SAP 도움말 포털에서 관련 문서와 지원을 찾아볼 수 있습니다.

설정하는 데 10분 정도밖에 안 걸리지만, 엄청난 시간을 절약할 수 있습니다. SAP를 정기적으로 사용한다면 꼭 설정하세요.

유니버설 ID는 시스템 변경이나 새로운 기능 도입 시 진가를 발휘합니다. [SAP 구축](https://noeldcosta.com/ko/creating-an-effective-sap-project-steering-committee/)모든 서비스에 새 로그인 계정을 만들 필요 없이, 모든 서비스를 유니버설 ID에 연결하세요. 모두에게 더 편리합니다.

하지만 유니버설 ID는 로그인에는 도움이 되지만 권한은 변경하지 않는다는 점을 기억하세요. 각 계정은 고유한 접근 권한을 유지합니다.

경험상, 얻는 것은 [SAP](https://noeldcosta.com/ko/sap-ecc-to-s4hana-migration-case-study/) 체계적으로 정리된 사용자 ID는 큰 효과를 가져옵니다. SAP를 일주일에 한 번 사용하든 매일 하루 종일 사용하든 모든 작업이 훨씬 원활해집니다.


궁금한 점이 있거나 현재 겪고 있는 상황에 대해 논의하고 싶으시면 언제든지 문의해 주세요. [SAP 구현](https://noeldcosta.com/ko/sap-license-negotiation-10-key-points-to-consider-in-2024/)궁금한 점이 있으면 언제든지 편하게 연락 주세요!

## 궁금하실 만한 질문들...

### 1\. SAP 유니버설 ID란 무엇이며, 왜 사용해야 할까요?

SAP Universal ID는 모든 SAP 계정을 연결하는 단일 로그인입니다. 이를 통해 서로 다른 비밀번호와 사용자 이름을 관리할 필요 없이 여러 SAP 시스템에 액세스할 수 있습니다.

### 2\. P-User 계정과 S-User 계정을 모두 SAP Universal ID에 연결할 수 있습니까?

네, 개인 P 사용자 계정과 회사 S 사용자 계정 모두 유니버설 ID에 연결할 수 있습니다. 로그아웃했다가 다시 로그인하지 않고도 계정 간 전환이 매우 간편해집니다.

### 3\. 제가 SAP 유니버설 ID를 생성하면 회사에서 알게 되나요?

유니버설 ID 생성은 개인적인 작업이며 회사와는 별개입니다. 하지만 회사 S-User를 연결하더라도 SAP 관리자는 해당 활동을 확인할 수 있습니다.

### 4\. 제가 회사를 떠나면 SAP Universal ID도 사라지나요?

아니요, SAP 유니버설 ID는 그대로 유지됩니다. 회사 S-Users 계정에는 접근할 수 없게 되지만, 유니버설 ID와 모든 P-Users 계정은 여전히 본인 소유입니다.

### 5\. SAP 유니버설 ID의 이메일 주소를 변경할 수 있나요?

네, 계정 설정으로 이동하여 업데이트하시면 됩니다. 이메일 주소를 변경한 후에는 반드시 새 이메일을 인증해야 합니다. 그렇지 않으면 계정이 잠길 수 있습니다.

### 6\. SAP 유니버설 ID에 몇 개의 계정을 연결할 수 있나요?

실질적인 제한은 없습니다. 저는 여러 클라이언트의 S 사용자 계정과 몇 명의 P 사용자 계정을 모두 제 계정에 연결했습니다.

### 7\. 연결된 계정의 비밀번호가 만료되면 어떻게 되나요?

S 사용자 계정의 비밀번호가 만료된 경우, 회사 SAP 관리자에게 문의해야 합니다. P 사용자 계정의 경우, 비밀번호 재설정 기능을 사용하십시오.

### 8\. SAP Universal ID를 사용하면 SAP 시스템에서 더 많은 권한을 얻을 수 있습니까?

전혀 그렇지 않습니다. 단지 로그인 과정을 간소화해 줄 뿐입니다. 실제 권한은 각 계정별로 다릅니다.

### 9\. SAP Universal ID에서 계정을 연결 해제하려면 어떻게 해야 하나요?

계정 설정으로 이동한 다음 연결된 계정을 찾아 연결 해제를 클릭하세요. 아주 간단합니다.

### 10\. SAP 시스템에 접속하려면 SAP Universal ID가 필수인가요?

선택 사항이지만, 정말 해볼 만한 가치가 있습니다. 원한다면 개별 자격 증명을 계속 사용할 수도 있지만, 굳이 더 번거롭게 만들 필요가 있을까요?
