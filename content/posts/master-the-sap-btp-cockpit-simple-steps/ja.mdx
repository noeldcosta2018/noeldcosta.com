# SAP BTP コックピットをマスターする: 誰でも実行できる簡単な手順!


### ノエル・D・コスタ

*   最後の更新 ： 2025 年 3 月 27 日

SAP Business Technology Platform (BTP) コックピットは、SAP クラウドサービスのコントロールセンターです。アプリケーション、サービス、セキュリティ設定、サブスクリプションを一元管理できます。

私は多くのクライアントとBTP導入に携わってきました。Cockpitは、SAPクラウドソリューションを運用するすべての人にとって日常的なコマンドセンターとなり、環境全体を直接制御できます。

BTPコックピットは主に3つのグループで使用されます。技術管理者は設定と監視を担当し、開発者はアプリケーションの導入と管理を行います。ビジネスユーザーは必要なツールにアクセスします。各役割はコックピットの異なる部分を使用します。

多くの企業はSAPクラウド製品を購入しても、コックピットの使い方をきちんと習得していないため、その価値を十分に享受できていません。本来は簡単なはずの基本タスクでさえ、苦戦しているのです。

BTP コックピットの使い方がわかれば、次のことが可能になります。

*   アプリケーションの実行速度を向上
*   クラウドコストを追跡および管理する
*   適切なセキュリティ権限を設定する
*   つながり、 [SAPを他のシステムへ](https://noeldcosta.com/ja/5-best-crm-systems-for-sap-in-2024/)

このガイドでは、 [SAP BTP コックピットの基本から高度な機能まで](https://noeldcosta.com/ja/sap-bpc-features-deployment-best-practice-guide/)これらは実用的である [SAPを改善するためにすぐに使える手順](https://noeldcosta.com/ja/sap-implementation/business-one/) クラウドのセットアップ。

BTP コックピットのナビゲーションと管理の基本から始めましょう。

SAP BTPコックピットの接続オプション（クラウドコネクタと宛先）により、驚くほど簡単に [クラウドアプリケーションをオンプレミスのSAPシステムに接続](https://noeldcosta.com/ja/start-your-sap-implementation-project-right/) およびサードパーティのソリューション。  

この統合レイヤーにより、BTPは単なるクラウドプラットフォームからコアの真の拡張機能へと変わります。 [SAP](https://noeldcosta.com/ja/system-implementation-sap/) 風景です。

## SAP BTP コックピットの 10 の重要なポイント

1.  **まずアカウント構造を計画します。** 戦略も立てずにサブアカウントをランダムに作成して苦戦する企業を見てきました。1日かけて [構造をマップする](https://noeldcosta.com/ja/2024-sap-timeline-planning-implementation-guide-essentials/) 飛び込む前に。後で感謝することになるよ。
2.  **権利付与モデルをマスターします。** グローバルアカウントには配布できるリソースのプールが限られています。私は最小限から始めて、必要に応じてスケールアップします。このアプローチは [未使用のサービスでクライアントに数千ドルを節約](https://noeldcosta.com/ja/sap-negotiation-advisors-reduce-cost/).
3.  **[ID プロバイダーに接続します。](https://noeldcosta.com/ja/create-and-manage-sap-universal-id-link-s-user-and-p-user-ids/)** SAP BTP コックピットは、会社の中央ユーザー管理にリンクすると最も効果的に機能します。私は最初にこれを設定しました。これにより、後々の多くの悩みが解消されます。
4.  **サービス マーケットプレイスは、最初は圧倒されます。** しかし、どこから始めればよいか分からない場合はどうすればよいでしょうか？ 基盤サービス、つまり宛先サービス、承認管理、 [クラウドコネクタ](https://noeldcosta.com/ja/sap-vs-oracle-which-erp-is-better-for-your-business/).
5.  **ランタイム環境を慎重に選択してください。** この決定はチームのスキルに基づいて行う必要があります。開発者が新しい環境を学習しなければならなかったために、プロジェクトが数か月遅れた例を私は見てきました。
6.  [統合機能は隠れた宝石です。](https://noeldcosta.com/ja/start-your-sap-implementation-project-right/) Cloud Connector と送信先を組み合わせると、システムの接続が驚くほど簡単になります。これが BTP が真価を発揮するところです。
7.  **必要になる前に監視を設定してください。** 重要なサービスのアラートをコックピットで設定します。毎朝コーヒーを飲みながらダッシュボードをチェックするのが、災難を防ぐための習慣になっています。
8.  **自動化ツールについて学びます。** BTP CLI と API は、最初は余分な作業のように思えますが、反復的なタスクにとっては画期的なものです。展開を自動化し、セットアップ時間を 80% 短縮しました。
9.  **カスタム ドメインにより、ユーザーの採用率が向上します。** セキュリティ セクションで時間をかけて設定してください。プロフェッショナルな外観は、エンド ユーザーにとって大きな違いをもたらします。
10.  **プラットフォーム外のすべてを文書化します。** コックピットはあなたの決定や推論を追跡しません。私はシンプルな Wiki を維持しており、これによってトラブルシューティングに費やす時間を数え切れないほど節約しています。

結局のところ、もしあなたが [SAP BTP](https://noeldcosta.com/ja/what-consultants-actually-do-beyond-using-buzzwords/) コックピットさん、あなたは全てを必要以上に難しくしています。これらの基本をマスターすれば、会社に必要なヒーローになれるでしょう。

## SAP BTPがあなたに提供するもの

[SAPビジネス](https://noeldcosta.com/ja/sap-clean-core-strategy-what-it-means-for-your-business/) テクノロジープラットフォームは、企業がクラウドに移行する際に直面する特定の問題に対処します。システムの接続、アプリケーションの構築、そして新しいテクノロジーの活用を支援します。

BTPは、SAPアプリケーションの拡張と異なるシステムの接続の基盤として機能します。さまざまなツールを1か所に統合することで、企業は適切な管理を維持しながらビジネス上の問題を解決できます。

主な課題を見てみましょう [SAP BTPは問題解決に役立ちます](https://noeldcosta.com/ja/sap-quality-gates-implementation/) その機能を備えています。

### 1\. 統合の課題

*   [SAP BTPはクラウドに接続します](https://noeldcosta.com/ja/sap-implementation/grow-with-sap/) Cloud Integration（旧CPI）を通じて、オンプレミスシステムとの連携を実現します。このサービスは、統合フローを作成するための視覚的なツールを提供します。
*   このプラットフォームは、複雑なコーディングなしで、S/4HANA、SuccessFactors、Ariba、Salesforce などの SAP 以外のシステムを接続します。
*   API 管理ツールは、ビジネス機能を他のアプリケーションに公開する API の作成と管理に役立ちます。
*   [オープンコネクタは事前に構築された統合を提供します](https://noeldcosta.com/ja/sap-implementation/rise-with-sap/) SAP 以外の何百ものサードパーティ システム向け。
*   Event Mesh を使用すると、ランドスケープ全体のビジネス イベントに基づいてシステムが通信できるようになります。
*   統合アドバイザーは、フィールド マッピングの提案を使用して、異なるシステム間でデータをマッピングするのに役立ちます。

### 2\. カスタム開発ニーズ

*   ABAPクラウド環境を利用すると、ABAP開発者はコアSAPコードを変更することなく拡張機能を作成できます。これにより、アップグレードパスが明確になります。
*   Cloud Foundry ランタイムは、Web アプリケーション開発のために Java、Node.js、Python と連携します。
*   Business Application Studioは、開発者に完全なWebベースの開発環境を提供します。ブラウザがあれば、あらゆるデバイスから作業できます。
*   [SAP Buildはビジネスに](https://noeldcosta.com/ja/sap-business-one-price-guide/) ユーザーは、ローコード ツールを使用して、高度な技術的スキルを必要とせずにシンプルなアプリケーションを作成できます。
*   拡張アプローチにより、コアシステムをクリーンな状態に保ちながらカスタマイズが可能になります。これにより、後々のアップグレードに伴う問題を軽減できます。
*   開発者は既存のスキルを活用しながらクラウドアプローチを学ぶことができます。このプラットフォームは、従来の開発と最新の開発の両方に対応しています。

### 3\. データ管理の複雑さ

*   データインテリジェンスクラウドは、データを移動および変換するためのパイプラインを作成します [システム間の情報](https://noeldcosta.com/ja/my-journey-with-customer-information-solutions-defense/)これにより、手動によるデータ転送が不要になります。
*   [HANAクラウドは](https://noeldcosta.com/ja/sap-testing-validation-tools-comparison/) アプリケーションと分析のためのデータベース機能。インメモリテクノロジーは高速に動作します。
*   Datasphere（旧称Data Warehouse Cloud）は、複数のソースからデータを統合し、ビジネスレポートを作成します。ビジネスユーザーは独自のデータモデルを作成できます。
*   あらかじめ構築されたコネクタにより、 [SAP以外のデータ統合](https://noeldcosta.com/ja/sap-testing-validation-tools-comparison/) ソースの参照が容易になり、異なるシステムを接続する際の時間を節約できます。
*   このプラットフォームは、リアルタイム、バッチ、イベントベースのデータ処理に対応しており、企業はそれぞれの状況に応じて適切なアプローチを選択できます。
*   データ品質管理ツールは、システム全体で情報の正確性を維持します。これにより、レポートと分析に対する信頼性が高まります。

## セクション 1: SAP BTP コックピットの使用開始


まずは基本から始めましょう。SAP BTPコックピットへのアクセスです。簡単そうに聞こえますが、ここで多くの人がミスを犯すのには驚かれるでしょう。

*   ちょうど行き **[https://account.hana.ondemand.com/](https://account.hana.ondemand.com/)** SAPのユーザー名とパスワードでログインしてください。これが他のすべての機能への入り口となります。 まだこれらをお持ちでない場合は、SAP 管理者に問い合わせて適切なアクセスを設定する必要があります。
*   グローバルアカウント構造は、ログイン後に最初に目にするものです。これは基本的に、会社のすべての情報を格納するメインコンテナです。 [SAPクラウド関連](https://noeldcosta.com/ja/sap-vs-oracle-which-erp-is-better-for-your-business/)私はいつもクライアントに、サブアカウントがすべて異なる部門である会社の本社のように考えるように言っています。
*   インターフェースは一見すると少し複雑に感じるかもしれません。左側にはナビゲーションメニューがあり、すべてのオプションが表示されます。メインエリアには、アカウント全体の概要と、各セクションのタイルが表示されます。 
*   上部には検索バーもあり、特定の設定を探すときに本当に多くの時間を節約してくれます。

しかし、SAP BTP コックピットの用語をまったく知らない場合はどうすればよいでしょうか。その場合は、迷子にならないようにするために、次の基本事項を知っておく必要があります。

*   **グローバルアカウント:** あらゆるものを入れるメインコンテナ
*   **サブアカウント:** 部門やプロジェクトごとに環境を分ける
*   **権利:** 利用が許可されているサービスとリソース
*   **サービスマーケットプレイス:** さまざまなSAPサービスを有効にする場所
*   **スペース：** 実際のアプリケーションがサブアカウントで実行される場所

最初のサブアカウントの設定は実はとても簡単です。グローバルアカウントビューから「サブアカウントを作成」ボタンをクリックしてください。 

次に、名前を付け、ドロップダウンからプロバイダー（AWSやAzureなど）を選択し、リージョンを選択します。パフォーマンスに大きな違いが出るため、ユーザー数の多い地域に最も近いリージョンを選択することをお勧めします。


作成後、新しいサブアカウントがグローバルアカウントの概要に表示されます。クリックすると、サービスの設定やアプリケーションのデプロイといった実際の作業を開始できます。

つまり、基礎を理解していないと、SAP BTP の他のすべてが、必要以上に難しくなってしまうということです。 

初めてこれを理解するのに何日もかかりましたが、あなたはそうする必要はありません。以下の手順に従うだけで大丈夫です。 [SAP BTP コックピットの操作](https://noeldcosta.com/ja/system-implementation-sap/) あっという間にプロのように。

## セクション 2: サブアカウントと権限の管理


話しましょう [SAP BTP コックピットでサブアカウントを整理する](https://noeldcosta.com/ja/create-and-manage-sap-universal-id-link-s-user-and-p-user-ids/)後々問題になるような混乱を招いている企業を数多く見てきました。私の場合は、開発、テスト、本番環境それぞれにサブアカウントを分けて作成しています。 

グローバルアカウントページから「サブアカウントを作成」をクリックすると作成できます。それぞれのアカウントに「マーケティング開発」や「財務製品」など、わかりやすい名前を付けて、何が何なのかを明確にしましょう。

しかし、複数の部署やプロジェクトがある場合はどうでしょうか？その場合は、作業を始める前に一貫した命名規則を作成する必要があります。 

これを信じてください...私はかつて、ランダムに名前が付けられたサブアカウントが 50 個以上あるクライアントと仕事をしたことがありましたが、誰も何も見つけられませんでした。

SAP BTPコックピットで特に興味深いのは、権限の割り当てです。基本的には、どのサブアカウントがどのサービスにどれだけアクセスできるかを決定します。 

グローバルアカウントの「Entitlements（エンタイトルメント）」セクションに移動してください。そこから「Configure Entitlements（エンタイトルメントを設定）」をクリックし、サブアカウントを選択して必要なサービスを追加してください。


エンタイトルメントモデル自体は非常にシンプルです。グローバルアカウントには、SAPから購入したエンタイトルメントのプールが存在します。 

必要に応じてサブアカウントに分配してください。最初は最小限の権限から始め、必要に応じて追加していくことをお勧めします。多くのサービスでは無料枠が用意されており、テストに最適です。

複数環境のデプロイメントについては、私が苦労して学んだベスト プラクティスをいくつか紹介します。

*   グローバル展開している場合は、サブアカウント構造を地域間でミラーリングします。
*   コストを管理するために開発権限を最小限に抑える
*   パフォーマンス関連のリソースを本番環境に優先させる
*   [権限の配布をSAP以外の場所に文書化します](https://noeldcosta.com/ja/best-sap-documentation-tools-2024-guide/) BTPコックピット
*   ラベルを使用して関連するサブアカウントを整理する

予期せぬ請求を避けたいなら、消費量を監視することは不可欠です。各サブアカウントの「使用状況分析」にアクセスして、実際に使用しているものと実際に支払っているものを比較してみましょう。私はすべてのクライアントについて、毎月この設定を確認しています。 

価値を提供せずに割り当てを消費しているサービスがいかに頻繁に見つかるかに驚かれることでしょう。

つまり、サブアカウントと権限を適切に管理しないと、お金を無駄にし、後で作業を遅らせる技術的負債を生み出すことになります。 

これを最初から正しく行うことで、SAP BTP 環境内のすべてがよりスムーズに実行されるようになります。


#### SAP BTP でサブアカウントと権限を管理する方法

管理エリア

内容

アクションステップ

サブアカウントを作成する

サブアカウントは、異なるプロジェクトまたはステージのグローバル アカウント内で分離された環境として機能します。

BTP コックピットの場合: 「グローバル アカウント」→「サブアカウントの作成」に移動して、「リージョン、環境、名前を選択」します。

権限の割り当て

エンタイトルメントは、サブアカウントで利用できるサービス (Integration Suite、HANA Cloud など) を定義します。

グローバル アカウント → 権限 → 編集 → サービス プランの追加 → サブアカウントへのクォータの割り当て。

サービスプランの管理

サブアカウントのニーズと使用モデルに基づいて、適切なプラン (無料、標準、開発) を選択します。

「権利」→「サービス別にフィルタリング」→「サブアカウントごとに割り当てる」で利用可能なプランを確認します。

クォータ使用量の追跡

消費量を監視することで、過剰使用や割り当て不足によるサービスの中断を防止できます。

「権限」→「クォータ使用率ダッシュボード」に移動し、サブアカウント別の割り当てと使用状況を確認します。

役割と信頼の設定

各サブアカウントには、アクセスを制御するためのロールコレクションと ID プロバイダー (IdP) が必要です。

サブアカウント → セキュリティ → 信頼構成とロールコレクション → ロールをユーザー/グループにマップします。

環境割り当て

サブアカウント内で使用されるランタイム (Cloud Foundry、Kyma、ABAP) を定義します。

サブアカウントを作成するときは、環境を選択 → それに応じて Cloud Foundry または Kubernetes を有効にします。

サービスの有効化

権限が割り当てられている場合でも、使用する前にサービスを有効にする必要があります。

サブアカウント → サービスマーケットプレイス → サービスを選択 → プランの有効化 → インスタンスの構成。

ラベル付けと組織化

ラベルは、ビジネス ユニット、地域、または目的別にサブアカウントを整理およびフィルタリングするのに役立ちます。

サブアカウントを作成/編集するときにタグを使用する → コックピット検索ツールを使用してフィルタリングします。

## セクション3: ユーザー管理と認証

ユーザー [SAPにおける管理](https://noeldcosta.com/ja/sap-ariba-your-2025-guide-to-sourcing-supplier-management/) BTPコックピットは、正しく使いこなさないと本当に頭が痛くなることがあります。私は苦労してこのことを知りました。 [最初の実装時の方法](https://noeldcosta.com/ja/erp-implementation-kpis-metrics/). 

まず、サブアカウントの「セキュリティ」セクションに移動し、「ユーザー」をクリックします。ここから、ユーザーを1人ずつ追加することも、CSVファイルをアップロードして一括追加することもできます。 

ユーザーが数人を超える場合は、一括アプローチを常にお勧めします。

しかし、組織に Azure AD や Okta などの ID プロバイダーがすでにある場合はどうでしょうか? その場合、ユーザーを手動で作成する必要はまったくありません。 

代わりに、IDプロバイダーとSAP BTP Cockpit の間に信頼関係を構築することをお勧めします。これは本当に時間の節約になります。

認証については、BTPコックピットにいくつかのオプションがあります。デフォルトはSAP IDサービスですが、エンタープライズ環境ではほとんど使用しません。 

カスタムSAML IDプロバイダーを使用すると、最も柔軟性が高まります。「セキュリティ」>「信頼設定」に移動して設定してください。SAPとIDプロバイダー間でメタデータファイルを交換する必要があります。


ロール コレクションは興味深いところです。これは基本的に、ユーザーに割り当てる権限のバンドルです。これが私のアプローチです:

*   個人ではなく職務に基づいてロールコレクションを作成する
*   具体的な内容にするには細分化しますが、数百件に及ぶほど細分化しないでください。
*   緊急時のシナリオに備えて、フルアクセス権を持つ緊急時の「管理者」ロールコレクションを含める
*   各ロールコレクションで何ができるかをチームが簡単に参照できる場所に文書化します。
*   役割の割り当てを四半期ごとに確認する。従業員は転職しても、古い権限が保持されることが多い。

SAP BTPコックピットでは、セキュリティのベストプラクティスが不可欠です。私は常に二要素認証を有効にし、可能な限り信頼できるネットワークへのアクセスを制限し、適切な監査ログを設定しています。 

また、「セキュリティアラート」セクションを定期的に確認する必要があります。SAP はそこで潜在的な問題について通知します。

つまり、BTP コックピットでユーザー管理と認証を正しく行わないと、将来的にセキュリティ上の問題や管理上の悪夢に直面することになります。 

早めに時間をかけてこれを正しく行えば、後で自分に感謝することになるでしょう。

## セクション 4: サービス インスタンスとアプリケーションの操作


SAP BTP Cockpit でのサービスインスタンスの作成は、何度か経験すれば簡単に思える作業の一つです…しかし、初めて使うと迷路のように感じるかもしれません。初めて試した時は、適切な場所を見つけるまでにすべてのメニューをクリックする必要がありました。

#### 私が今やっているやり方は次のとおりです（時間の無駄なし）：

1.  サブアカウントで、 **「サービス」→「サービスマーケットプレイス」**

2.  利用できるすべてのサービスが表示されます。

3.  希望するものをクリックして、 **"作成する。"**

4.  プランを選択してください（これにより、提供されるサービスと支払う金額が決まります）。

5.  求められる設定を入力します。私はいつもこれらの設定を安全な場所に保存しています。後で何か問題が起きたときの救世主となるからです。

#### サービスをアプリにリンクする:

作成した後も、アプリはその存在を認識しません。そこで **サービスバインディング** お入りください。

*   アプリへ移動 → **「サービスバインディング」** → サービス用に新しいものを作成します。

*   BTP は接続と資格情報を処理します。

#### サービス キー (私のお気に入りのトリック):

これらは、BTP 外部のツールやユーザーのための特別なログインのようなものです。

*   サービスを探す → **「サービスキー」** → **"作成する。"**

*   覚えやすい名前を付けましょう。 `dev-team-access` or `jenkins-deployment`.

#### 忍耐力を失わずにサービス マーケットプレイス内を移動するには:

*   探しているものが分かっている場合は、検索バーを使用してください。

*   その **「最近使用した」** セクションは、何度も戻ってくるものに対して便利です。

*   探索するときにフィルターが役立ちます。

*   テストだけなら、 **Free** 計画。

*   サービスが自分の環境（Cloud Foundry、Kyma など）で動作するかどうかを確認します。

#### サービスを監視します:

すべてのサービスには **"モニタリング"** セクションを参照してください。重要なものがないかどうか毎週チェックしています。  
次の内容が表示されます:

*   応答速度

*   エラーの頻度

*   どれだけの資源を使っているか  
    中には、追加情報を含む詳細なダッシュボードを備えたものもあります。

#### 結論：

サービスインスタンスを追跡していないと、BTPのメリットの半分を見逃していることになります。真の価値はここにあります。一度使い方を覚えてしまえば、プラットフォーム全体がより使いやすくなります。

## セクション 5: SAP BTP へのアプリケーションの導入


SAP BTPへのアプリケーションの導入は、最初は難しそうに思えるかもしれません。コックピットに表示されるたくさんのオプションを見つめ、どこから始めればいいのか途方に暮れたのを覚えています。 

幸いなことに、SAP では複数の導入オプションが提供されており、特定のニーズに最適なものを選択できます。

SAP BTP Cockpit に表示される主なデプロイメントオプションは、Cloud Foundry、Kyma、ABAP の 4 つです。私は通常、ほとんどの Web アプリケーションに Cloud Foundry を使用し、Kubernetes 機能が必要な場合は Kyma を使用し、S/XNUMXHANA の拡張には ABAP を使用しています。 

コックピットにはそれぞれ独自のセクションがあるため、使用したいものを有効にする必要があります。

しかし、どの環境を選択すればよいか分からない場合はどうすればよいでしょうか？まずはチームのスキルを考慮してください。開発者がABAPに精通している場合は、ABAP環境を使い続けることをお勧めします。 

コンテナとマイクロサービスに精通している場合は、Kyma の方が適している可能性があります。

基本的な Cloud Foundry の導入手順を説明します。まず、サブアカウントに Cloud Foundry スペースを作成したことを確認します。次に、次の手順を実行します。

*   SAP BTPコックピットのスペースに移動する
*   「アプリケーションのデプロイ」をクリックします
*   アプリケーションファイル（通常はJARまたはWARファイル）をアップロードします。
*   アプリケーション名やその他の基本情報を設定します
*   メモリ割り当てとインスタンスを構成する
*   デプロイを押して魔法が起こるのを待ちます

アプリケーションのバージョン管理については、公式ガイドでは十分に説明されていません。メジャーバージョンごとに個別のルートを作成し、更新にはブルーグリーンデプロイメントを使用します。これにより、本番トラフィックを切り替える前に新しいバージョンをテストできます。

環境変数は設定において非常に重要です。BTP Cockpit で設定するには、デプロイしたアプリケーションに移動し、「環境変数」をクリックしてキーと値のペアを追加します。 

APIエンドポイント、機能フラグ、機密性が低い設定などはここに保存します。パスワードやトークンは環境変数には絶対に入れず、代わりに宛先サービスを使用してください。

コックピットではスケーリングが簡単です。アプリケーションの「リソース」セクションで、インスタンス数とメモリ割り当てを手動で調整できます。 

実稼働アプリの場合、高可用性を確保するために少なくとも 2 つのインスタンスを推奨します。

つまり、SAP BTP コックピットのデプロイメント オプションを理解していない場合、チームが構築できる内容が制限されることになります。 

時間をかけてこれらのデプロイメント パターンを学習すれば、プラットフォームがほぼあらゆる状況に対応できることがわかります。

## セクション 6: 統合と拡張のシナリオ


[SAPの統合シナリオ](https://noeldcosta.com/ja/why-sap-integrated-business-planning-sap-ibp-matters/) BTP Cockpit は、パターンを理解するまでは戸惑うほどでした。おそらく、S/4HANA システムへの接続が目的でしょうし、正直言って、その努力は価値があります。 

コックピットでは、まず、BTP 環境と社内システム間の通信を安全にトンネリングする小さなオンプレミス コンポーネントである Cloud Connector を設定する必要があります。

Cloud Connector をインストールしたら、SAP BTP Cockpit にログインし、サブアカウントに移動して、左側のメニューで「接続」を探します。 

ここでクラウドコネクタを追加し、アクセスできる社内システムを定義する必要があります。私は常に最小限のアクセスから始め、必要に応じて拡張していきます。

しかし、オンプレミスのシステムではなくクラウドシステムに接続しようとしている場合はどうでしょうか？その場合は、Cloud Connector ではなく宛先を使用します。 

宛先とは、基本的にBTPコックピットに保存される接続設定です。「接続」>「宛先」に移動して、新しい宛先を作成してください。URL、認証タイプ、その他のプロパティを指定する必要があります。

SAP BTPコックピットで適切な宛先を設定することは非常に重要です。私は常に以下の点に重点を置いています。

*   システムと目的を示す明確な命名規則を使用する
*   資格情報はコード内ではなく宛先に保存する
*   異なる環境（開発/テスト/本番）ごとに別々の宛先を設定する
*   プロパティを使用してタイムアウト値やその他の接続パラメータを制御する
*   開発者に頼らせる前に目的地をテストする

API管理は、BTP Cockpit の強力な機能の一つです。実際のエンドポイントの前にAPIプロキシを作成することで、レート制限、セキュリティ、変換を制御できます。 

パートナーや顧客に公開されるすべての API にこれを常に使用します。

より複雑な統合シナリオについては、 [統合スイート](https://noeldcosta.com/ja/sap-integration-suite-delivery-delays/) サービスマーケットプレイスで。 

これにより、クラウド統合、API 管理、オープン コネクタなどのツールをすべて BTP コックピットから管理できるようになります。

つまり、SAP BTP コックピットの統合機能を本当に理解していないと、データを効果的に共有できないサイロ化されたシステムに固執することになり、間違いなく問題が発生することになります。

## セクション 7: 監視と分析


**SAP BTPコックピットでの監視** 仕事の中で最も刺激的な部分ではありませんが、不意打ちを食らうことを防ぐ手段です。多くのチームがこれを無視し、必ず後で痛い目に遭うのを見てきました。

**どこから始めれば：**

*   に行きます **"モニタリング"** サブアカウントのセクション。

*   すべてのアプリとサービスのスナップショットと、現在のステータスが表示されます。

*   私は毎朝これをチェックしています。これにより、「なぜシステムがダウンしているのか？」という瞬間を何度も回避できました。

**アラートの設定:**

*   アプリ内で **「監視」→「アラートの設定」**.

*   次のアラートを設定できます:

    *   遅い応答時間

    *   エラー率が高い

    *   リソースの使用量が多い

*   私がいつも使っている設定: 重要なアプリにはメールアラート、ダウンできないアプリには Slack Webhook。

**使用状況を監視中:**

*   へヘッド **「使用状況分析」**これは、リソースがどこに割り当てられているかを確認する最も簡単な方法です。

*   それは次のように分類されます:

    *   サービス

    *   サブアカウント

    *   期間

*   無駄をキャッチし、コストの問題になる前に最適化を行うためにこれを使用します。

**クライアントのコストを抑える方法:**

*   使用していないサービスを毎月削除する

*   営業時間外に開発環境をスケールダウンする

*   サービス プランを適切な環境 (開発環境と本番環境) に適合させる

*   使用量の急増に注意してください。これは通常、何かがおかしい兆候です。

*   可能な場合は無料プランのサービスを利用する

**トラブルシューティング：**

*   Cloud Foundryアプリの場合: アプリを開く → **「ログ」**

*   サービスについては、 **"モニタリング"** そのサービスインスタンスのセクション

*   ほとんどの問題はここに手がかりを残しているので、推測する前に必ず確認してください。

**ボトムライン：**  
BTP環境を監視していないと、状況を把握できていないのと同じです。小さな問題があっという間に大きな問題に発展し、コストが手に負えなくなる可能性があります。常に監視することで、状況をコントロールできます。

## セクション 8: 高度な BTP コックピット機能


私が苦労して学んだSAP BTPコックピットの高度な機能をいくつかご紹介します。きっと役立つはずです。 [時間を節約](https://noeldcosta.com/ja/how-to-create-an-sap-implementation-project-charter/) 一度マスターしてしまえば、イライラすることはありません。

約1年前から、繰り返しのタスクを自動化するためにBTP APIを使い始めました。サブアカウントの作成、サービスのプロビジョニング、ユーザーの割り当てなど、ほぼすべてのタスクをスクリプト化できます。 

先週、12人の新しい開発者のために環境を構築する必要がありました。一日中コックピットを操作する代わりに、スクリプトを実行してコーヒーを飲みに行きました。戻ってきたら、すべて準備が整っていました。

でも、ブラウザインターフェースが全く嫌ならどうすればいいでしょうか？そんな時は、BTPコマンドラインインターフェースをダウンロードしましょう。私は今、これを頻繁に使っています。コックピットの画面をあれこれ操作するよりも、簡単なコマンドを入力する方がずっと速いです。 

端末から直接アプリを展開し、ステータスを確認し、サービスを管理できます。

カスタムドメインはユーザーにとって大きな違いをもたらします。一般的なSAP URLのアプリを使いたい人は誰もいません。 

サブアカウントの「セキュリティ」>「カスタムドメイン」からアクセスできます。ドメインをTXTレコードで認証し、SSL証明書をアップロードする必要があります。 

私はクライアントのプロジェクトのためにこれをやったのですが、クライアントの幹部はすぐにプロフェッショナルな雰囲気が増したことに気づきました。

マルチリージョン展開の場合、次の方法が有効です。

*   絶対にダウンできないアプリケーションのためのアクティブ/アクティブ
*   データ保存要件に応じた地域固有の展開
*   完璧な稼働時間よりも予算が重要な場合のプライマリバックアップ設定
*   世界中で最高のユーザーエクスペリエンスを実現するグローバルルーティングレイヤー

パイプラインでサービス キーと CF CLI を使用して、SAP BTP Cockpit を DevOps ツールと統合しました。現在、開発者がコードをプッシュすると、誰も Cockpit に触れることなく自動的にデプロイされます。これにより、デプロイ時間が数時間から数分に短縮されました。

つまり、SAP BTP コックピットですべてを手動で行っている場合、貴重な時間を無駄にし、おそらくエラーが発生することになります。 

これらの高度な機能には多少の設定が必要ですが、BTP 環境の管理方法が変わります。

## ご質問があるかもしれません...

### 1\. SAP の BTP コックピットとは何ですか?

SAP [BTPコックピット](https://noeldcosta.com/ja/sap-enterprise-warehouse-management-sap-ewm-essentials/) SAP Business Technology Platform 環境を管理するための、Web ベースの集中管理インターフェースです。サブアカウントの設定、セキュリティの設定、アプリケーションの導入、サービスの管理など、さまざまな機能をご利用いただけます。SAP のクラウド プラットフォームで実行されるすべてのものを管理するコントロールセンターとしてご利用ください。

### 2\. BTP コックピットへはどうやって行けばいいですか?

SAP BTPコックピットにアクセスするには、 [https://account.hana.ondemand.com/](https://account.hana.ondemand.com/) SAP 認証情報を使用してログインします。まだアクセス権がない場合は、会社の SAP 管理者に適切な権限を設定するよう依頼する必要があります。

### 3\. SAP BTP の 4 つの柱は何ですか?

SAP BTP の 4 つの柱は次のとおりです。

*   データベースとデータ管理 – データの保存、処理、分析を行うツール
*   アプリケーション開発 – クラウドアプリケーションと拡張機能を構築するためのサービス
*   統合 – システムとプロセスを接続するためのソリューション
*   インテリジェントテクノロジー – AI、ML、分析機能

これらの柱は、BTP コックピットを通じて管理できるものの基盤を形成します。

### 4\. SAP のコックピット ツールとは何ですか?

SAP の Cockpit ツールは、さまざまな SAP クラウド ソリューションを管理するための Web ベースの管理インターフェイスです。特に SAP BTP の場合、Cockpit を使用すると、コマンドライン ツールや API をすべて使用しなくても、構成、展開、監視を視覚的に処理できます。

### 5\. SAP BTP トライアルアカウントの作成はどのように行われますか?

BTPトライアルアカウントの作成は非常に簡単です。 [https://account.hanatrial.ondemand.com/](https://account.hanatrial.ondemand.com/)をクリックし、「登録」をクリックして、手順に従ってください。リソースが制限された無料アカウントが 90 日間有効になりますが、延長することもできます。試用版では、使用制限はあるものの、ほとんどの BTP サービスにアクセスできます。

### 6\. SAP BTP コックピットの管理はどのように機能しますか?

SAP BTP Cockpit の管理は、ロールベースのアクセス モデルを通じて行われます。グローバル管理者はサブアカウントを作成し、管理者を割り当てることができます。各レベルには特定の責任があり、グローバル管理者は権限とクォータを処理し、サブアカウント管理者はサービスとアプリケーションを管理します。すべては、さまざまな管理タスクの明確なセクションを備えた Web インターフェイスを通じて行われます。

### 7\. SAP BTP コックピットにアクセスするにはどうすればいいですか?

SAP BTPコックピットにアクセスするには、 [https://account.hana.ondemand.com/](https://account.hana.ondemand.com/) SAPユーザーIDとパスワードでログインしてください。トライアルアカウントをご利用の場合は、 [https://account.hanatrial.ondemand.com/](https://account.hanatrial.ondemand.com/) 代わりに、まず組織の管理者からアクセス権を付与される必要があります。

### 8\. SAP BTP コックピットでユーザーを作成するにはどうすればよいですか?

SAP BTP コックピットでユーザーを作成するには:

1.  サブアカウントに移動する
2.  左メニューの「セキュリティ」セクションに移動します
3.  「ユーザー」を選択
4.  「作成」ボタンをクリック
5.  ユーザーの詳細とメールアドレスを入力してください
6.  必要に応じてロールコレクションを割り当てる
7.  変更を保存します

その後、ユーザーはアカウントを有効化するためのメールを受け取ります。

### 9\. SAP CPI BTP コックピットはどのように機能しますか?

BTP コックピットの SAP Cloud Integration (CPI) は、Integration Suite を通じて統合機能を提供することで機能します。コックピットでは、Integration Suite にサブスクライブし、Integration Suite UI にアクセスして統合フローを設計、展開、監視します。コックピット自体は、サブスクリプション、権限、およびセキュリティの側面を管理します。

### 10\. SAP BTP コックピットの URL は何ですか?

SAP BTPコックピットのメインURLは [https://account.hana.ondemand.com/](https://account.hana.ondemand.com/)トライアルアカウントの場合は、 [https://account.hanatrial.ondemand.com/](https://account.hanatrial.ondemand.com/)これらにより、グローバル アカウント レベルに移動し、特定のサブアカウントに移動できるようになります。

### 11\. SAP BTP コックピットへのアクセスはどのように機能しますか?

SAP BTP コックピットへのアクセスは、階層型の権限システムを通じて行われます。ユーザーには、コックピット内の特定のアクションと領域への権限を付与する特定のロール コレクションが割り当てられます。グローバル管理者はすべてにアクセスできますが、他のユーザーは割り当てられたロールに基づいて特定のサブアカウントまたはサービスにのみアクセスできます。

### 12.SAP BTP Cockpit グローバルアカウントとは何ですか?

SAP BTP Cockpit グローバル アカウントは、すべての SAP クラウド リソースの最上位コンテナです。ここで、権限の管理、サブアカウントの作成、サブアカウント間の問題の処理を行います。通常、会社には 1 つのグローバル アカウントがあり、その中にさまざまな部門、プロジェクト、または環境の複数のサブアカウントが含まれています。

### 13\. SAP BTPコックピットの概要を教えてください

SAP [BTPコックピット](https://noeldcosta.com/ja/sap-analytics-cloud/) クラウドリソースを管理するための一元的なインターフェースを提供します。機能は明確なセクションに整理されています。

*   アカウントエクスプローラー – アカウント間を移動します
*   セキュリティ – ユーザー管理と認証
*   サービス – クラウドサービスの有効化と管理
*   スペース – アプリケーションのデプロイメント用 (Cloud Foundry 内)
*   モニタリング – 使用状況とパフォーマンスを追跡するため
*   接続性 – 他のシステムとの統合

### 14\. SAP における BTP の目的は何ですか?

SAP BTP の目的は、SAP アプリケーションを拡張および統合するための統合クラウド プラットフォームを提供することです。これにより、カスタム アプリケーションの構築、さまざまなシステムの接続、ビジネス プロセスへのインテリジェンスの追加、データの管理が可能になり、コア SAP システムとの互換性も維持されます。BTP は、標準の SAP 機能と特定のビジネス ニーズとのギャップを埋めます。

### 15\. SAP BTP は AWS に似ていますか?

SAP BTP と AWS は目的が異なります。AWS は汎用クラウド インフラストラクチャ プロバイダーですが、SAP BTP は SAP アプリケーションの構築と拡張専用に設計されたプラットフォームです。BTP にはインフラストラクチャ コンポーネントが含まれていますが、その主な価値は、提供されるビジネス サービスと SAP 固有の統合機能にあります。多くの企業が実際に AWS などのインフラストラクチャ プロバイダー上で SAP BTP を実行しています。

### 16\. BTP の完全な形式は何ですか?

BTP は Business Technology Platform の略です。SAP は、クラウド インフラストラクチャだけにとどまらないビジネス重視のプラットフォームとしての包括的な性質をよりよく反映するために、これを SAP Cloud Platform (SCP) から改名しました。

### 17\. SAP BTP コックピットでユーザーを作成するにはどうすればよいですか?

SAP BTP コックピットでユーザーを作成するには:

1.  コックピットにログインし、サブアカウントに移動します
2.  左側のナビゲーションから「セキュリティ」をクリックします
3.  「ユーザー」を選択
4.  「作成」ボタンをクリックします
5.  ユーザーのメールアドレスと名前を入力してください
6.  ユーザーを保存する
7.  ユーザーに適切なロールコレクションを割り当てる

### 18\. プログラミングにおけるコックピットとは何ですか?

プログラミングおよびソフトウェア システムでは、「コックピット」は一般に、包括的な管理機能を提供する集中ダッシュボードまたは制御インターフェイスを指します。航空機のコックピットのように、複雑なシステムの操作に必要なすべてのコントロールとディスプレイが統合されています。SAP BTP コックピットは、プラットフォームのすべての管理機能を一元化することでこのコンセプトに従います。

### 19\. BTP を SAP に接続するにはどうすればよいですか?

BTP を既存の SAP システムに接続するには:

1.  オンプレミスネットワークにクラウドコネクタを設定する
2.  特定のシステムをBTPに公開するように設定する
3.  BTPコックピットで、サブアカウントの「接続」セクションに移動します。
4.  Cloud Connectorの接続を確認する
5.  SAPシステムを指す宛先を作成する
6.  BTPアプリケーションでこれらの宛先を使用する

Cloud Connector は、内部ネットワークと SAP BTP の間に安全なトンネルを提供します。
