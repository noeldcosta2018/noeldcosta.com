## So verwenden Sie das S/4HANA-Migrationsstrategietool

Die Anwendung des S/4HANA Migration Assessment Tools ist einfach, doch die Ergebnisse können Sie später vor kostspieligen Fehltritten bewahren. Es soll Ihnen helfen, eine Entscheidung zu treffen – Greenfield, Brownfield oder etwas dazwischen – basierend auf dem aktuellen Stand Ihres Systems und Ihren Zielen. So geht's:

#### 1\. **Systemalter (Jahre seit Inbetriebnahme)**

Wählen Sie zunächst aus, wie lange Ihr aktueller [SAP-System](https://noeldcosta.com/de/sap-implementation/) läuft. Ein neueres System (1–4 Jahre) könnte zu Brownfield tendieren, insbesondere wenn alles noch reibungslos funktioniert. Ein mittelaltes System (5–9 Jahre), wie in diesem Beispiel, kann zwar Ineffizienzen angesammelt haben, aber dennoch einen Wert besitzen, der erhaltenswert ist. Ältere Systeme (10+ Jahre) veranlassen Unternehmen oft dazu, einen saubereren Bruch in Betracht zu ziehen.

#### _**2\. Anzahl der benutzerdefinierten Objekte (Z Objekte)\***_

Benutzerdefinierter Code spielt bei der Migrationsplanung eine große Rolle. Wenn Sie 151–400 benutzerdefinierte Objekte haben, ist die Komplexität moderat. Genug, um relevant zu sein. Je mehr Z-Code Sie haben, desto schwieriger ist eine saubere Konvertierung, insbesondere wenn Sie nicht wissen, was aktiv verwendet wird. Brownfield kann diese Objekte zwar intakt halten, aber Greenfield ermöglicht Ihnen, alles zu bereinigen.

#### 3\. **Datenqualität**

Als Nächstes bewerten Sie Ihre Daten. Seien Sie ehrlich. Sind sie sauber? Konsistent? Wenn Ihre Antwort „einige Validierungen, einige Inkonsistenzen“ lautet, liegen Sie in der Mitte. Ihre Daten sind wahrscheinlich nutzbar, benötigen aber vor der Übertragung etwas Vorbereitung. Bei Greenfield müssen Sie die Daten möglicherweise trotzdem bereinigen. Bei Brownfield werden sie möglicherweise unverändert übertragen, was riskant sein kann. Das Tool berücksichtigt dies.

#### 4\. **Notwendigkeit einer Neugestaltung der Geschäftsprozesse**

Wenn Sie nur „einige Schlüsselbereiche“ neu gestalten, bedeutet das, dass Sie nicht alles überholen. In diesem Fall ist die selektive Transition eine echte Option. Sie ermöglicht es Ihnen, Probleme zu beheben, ohne das gesamte System über den Haufen zu werfen. Vollständige Neugestaltung? Sie tendieren zu Greenfield. Minimale Änderungen? Brownfield könnte ausreichen.

#### 5\. **Aufbewahrung historischer Daten**

Benötigen Sie wirklich jede Transaktion der letzten zehn Jahre? Oder nur die wichtigen Geschäftsunterlagen? Selektive Aufbewahrung bedeutet, dass Sie es sich leisten können, einen Teil der Vergangenheit zurückzulassen. Das eignet sich gut für Greenfield- oder selektive Transition, bei der Sie nicht alle historischen Daten in das neue System übernehmen. Brownfield bringt alles mit sich, sofern Sie es nicht filtern.

#### 6\. **Projektbudget (USD)**

Der letzte, aber nicht unwichtige Punkt ist Ihr Budget. Ein moderates Budget (1–5 Millionen US-Dollar) schließt eine vollständig individuelle selektive Transition oft aus, es sei denn, der Umfang ist sehr eng begrenzt. Sowohl Greenfield als auch Brownfield können in diesen Bereich passen, aber die Kontrolle des Umfangs ist entscheidend. Brownfield ist oft im Vorfeld günstiger; Greenfield kostet zwar zunächst mehr, spart Ihnen aber langfristig Wartungskosten. Das Tool wägt diesen Kompromiss ab.

Sobald Sie alle Parameter eingegeben haben, klicken Sie einfach auf **„Empfehlung berechnen“**Sie erhalten einen Strategievorschlag sowie eine Erklärung, die direkt auf Ihre Eingaben abgestimmt ist. Es ist keine Zauberformel, aber es bietet Ihnen einen fundierten Ausgangspunkt, dem Sie leichter vertrauen können als einem Bauchgefühl oder einem Verkaufsgespräch.

## Wie das Tool entscheidet

Das Tool wertet keine Antworten aus. Es verbindet die Punkte: was Sie haben, was Sie wollen und was schiefgehen könnte, wenn Sie den falschen Weg wählen.

*   **Systemzeitalter**  
    Ältere Systeme (10+ Jahre) weisen oft auf Greenfield hin. Aber wenn alles stabil und sauber ist, kann Brownfield immer noch funktionieren.

*   _Benutzerdefinierter Code (Z-Objekte)_\*  
    Eine hohe Anzahl an Anpassungen erhöht das Risiko. Brownfield behält sie bei; Greenfield gibt Ihnen die Möglichkeit zum Neustart.

*   **Datenqualität**  
    Schlechte oder inkonsistente Daten führen in der Regel dazu, dass die Empfehlung von Brownfield abweicht. Durch Migration werden die Daten nicht bereinigt, sondern nur verschoben.

*   **Prozessneugestaltung**  
    Wenn Sie nur wenige Bereiche anpassen, ist Brownfield die richtige Wahl. Bei größeren Änderungen tendiert das Tool zu Greenfield oder Selective.

*   **Datenerhaltung**  
    Benötigen Sie eine vollständige Historie? Brownfield ist einfacher. Selektive oder Greenfield erfordern mehr Filterung und Vorbereitung.

*   **Budget**  
    Ein begrenztes Budget schränkt die Optionen ein. Das Tool drängt nicht auf etwas, das Sie wahrscheinlich nicht gut umsetzen können.

Es geht nicht darum, Perfektion anzustreben, sondern nur um eine Richtung, die auf der Grundlage Ihrer aktuellen Situation Sinn ergibt.
